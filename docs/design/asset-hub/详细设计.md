## 1. 页面概述

- **路由**: `/wande/asset`
- **菜单层级**: 资源中心 > 方案素材中心
- **权限标识**: `wande:asset:*`
- **所属模块**: `module:fullstack`
- **核心表**: `assets` / `asset_versions` / `asset_tags` / `asset_project_relations` / `asset_download_log` / `asset_audit_log` / `watermark_rules` / `asset_recommendations` / `asset_embedding`
- **原型架构**: 5 Tab 多文件架构（`index.html` + 5 个 `tab-*.html`）
- **依赖服务**:
  - S3 对象存储（文件上传/下载/缩略图）
  - CLIP 微服务（AI 自动分类 + 向量 embedding 生成）
  - Spring ApplicationEvent 事件总线（中标状态→权限同步）
  - pgvector / Milvus（向量检索，视 MySQL 8.0 版本支持情况选型）
  - PPT 插件 Webhook（`/wande/ppt-plugin/insert`）

### 1.1 业务背景

万德方案素材长期散落在设计师本地硬盘、企业网盘和项目文件夹中，销售/设计师在客户提出"需要动物造型方案"时平均耗时 20 分钟检索素材，且缺乏权限管控（中标项目素材泄露给竞对风险）。方案素材中心的目标是：

1. **统一 DAM**（Digital Asset Management）：集中存储、全文检索、多维度筛选，实现"3 秒找到目标素材"。
2. **自动权限管理**：素材可见性与项目中标状态绑定，中标后自动扩大访问权限，避免人工维护 ACL 清单。
3. **AI 推荐**：根据项目类型/行业/风格智能推荐 TOP-30 素材，直接推送到 PPT，节省方案制作时间。
4. **全链路审计**：下载/版本/水印全记录，满足内部知识产权保护需求。

### 1.2 用户角色

| 角色 | 典型操作 | 权限范围 |
|------|---------|---------|
| 设计师 | 上传素材 / AI 分类 / 版本管理 | 上传 + 管理自有素材 + 浏览（权限范围内） |
| 销售/商务 | 浏览 + 下载 + AI 推荐 + PPT 插入 | 只读（权限范围内） |
| 项目经理 | 浏览 + 绑定项目 | 浏览 + 权限配置（项目维度） |
| 超管（吴耀） | 全部操作 + 权限矩阵管理 + 审计 | 全量 |

### 2.1 素材浏览（Tab1 — tab-browse.html）— 对应 Issue #1882

**功能描述**: 左侧分类树 + 顶部多维度筛选栏 + 卡片瀑布流主体 + 分页器。用户在此完成"找素材"的核心场景。

- **组件**:
  - `ATree`（左侧分类树，支持折叠，高亮选中节点）
  - `AssetFilterBar`（顶部筛选栏：资产类型 / 文件格式 / 场地类型 / 适龄群体 / 色彩风格 / 材质 / 关键字 / 中标状态）
  - `AssetCardGrid`（卡片瀑布流，支持卡片/列表双视图切换）
  - `AssetCard`（单卡：缩略图 + 名称 + 类型标签 + hover 浮层操作按钮）
  - `APagination`（底部分页，默认 20/页，支持 20/50/100 切换）
  - `AssetDetailDrawer`（右侧抽屉：素材详情 + 版本历史 + 权限标签）

- **数据源**:
  - `GET /wande/asset/list`（请求参数 `AssetBo`：`categoryId` / `assetType` / `fileType` / `venueType` / `ageGroup` / `colorStyle` / `materialType` / `keyword` / `bidStatus` / `pageNum` / `pageSize`）
  - `GET /wande/asset/category/tree`（分类树数据，支持懒加载）
  - `GET /wande/asset/detail?assetId=XX`（详情抽屉数据）

- **交互**:
  - 左侧分类树点击节点 → 触发 `categoryId` 筛选，右侧卡片区刷新
  - 顶部筛选栏各维度独立可选（多选下拉）→ 组合过滤，筛选条件同步到 URL query string（支持分享链接）
  - 关键字搜索 → 后端 `MATCH AGAINST` 全文索引（`name` + `description` + `tags`），防 SQL 注入
  - 卡片 hover 浮出 4 个操作按钮：下载（带水印）/ 预览（Lightbox）/ 收藏 / 对比（最多 4 张）
  - 视图切换（卡片/列表）→ 本地 localStorage 保持用户偏好
  - 分页器切换页码 → 滚动到顶部，URL hash `#page=N` 同步

### 2.2 上传分类（Tab2 — tab-upload.html）— 对应 Issue #1793 #2163

**功能描述**: 设计师将素材文件（图片/视频/PDF/3D 模型等）拖拽上传到平台，系统调用 CLIP 微服务自动预分类并打标签，设计师在编辑队列表格中修正后批量保存。

- **组件**:
  - `UploadDropzone`（拖拽上传区，支持批量选择，显示文件类型图标和大小预检）
  - `UploadProgressList`（上传进度列表，每文件独立进度条 + 状态标识）
  - `UploadQueueTable`（上传队列编辑表格：文件名/预分类结果/AI 标签/项目绑定/可见性/操作）
  - `ClassifyEditor`（AI 预分类结果可手工修正的 inline 编辑组件）
  - `ProjectBindSelect`（项目绑定多选，关联 `asset_project_relations`）
  - `BatchActionBar`（批量操作栏：批量保存 / 批量设置分类 / 批量删除队列项）

- **数据源**:
  - `POST /wande/asset/upload`（multipart/form-data，返回 `assetId` + `pre_category` + `pre_tags`）
  - `POST /wande/asset/bulk-update`（批量保存分类 + 标签 + 项目绑定）
  - `GET /wande/asset/ai-classify?assetId=XX`（轮询 AI 分类结果，CLIP 模型识别 + 文件名启发式，异步返回）
  - `GET /wande/project/select-list`（项目下拉选项，供绑定）

- **交互**:
  - 文件拖入 Dropzone → 前端预检文件大小（≤500MB）和类型（白名单）→ 开始上传
  - 上传成功后异步触发 AI 分类（后端推入队列），前端轮询 `/ai-classify` 直至状态为 `done`（轮询间隔 2s，最多 30 次）
  - AI 分类结果回填到 `UploadQueueTable`，用户可 inline 修正分类和标签
  - 绑定项目后，系统根据该项目 `bid_status` 自动推算初始可见性（`team_only` / `all`）并在表格显示
  - 全部编辑完成后点击「批量保存」→ 调用 `bulk-update` → 成功后清空队列，Toast 提示
  - 上传失败（网络/大小超限）：自动重试 3 次（exponential backoff），超过后标红并提示手动重试

- **代码现状**: 🟡 有基础需改造
  - S3 文件上传封装已存在（`StorageService`），可复用 presigned URL 生成逻辑
  - AI 分类为新能力，需新增 `ClipClassifyService` 对接 CLIP 微服务（HTTP 调用 + 异步任务队列）
  - 批量保存接口（`bulk-update`）需新建，关联写入 `assets` + `asset_project_relations` + `asset_audit_log`

- **已有文件**:
  - `backend/ruoyi-modules/wande-ai/.../service/StorageService.java`（S3 上传封装，可复用）

- **三级边界**:

```
✅ Always（必须做）:
- 单文件大小上限 500MB，前端预检 + 后端双重校验
- 文件类型白名单：jpg/jpeg/png/webp/gif/svg/mp4/mov/pdf/psd/ai/sketch/glb/fbx/obj（后端 MIME 验证，不信任扩展名）
- 上传失败自动重试 3 次（指数退避：1s / 2s / 4s），超过后暂停并标红
- 触发 AI 分类必须异步（推 MQ/异步任务，不阻塞上传响应）
- 所有上传操作写入 asset_audit_log（action=UPLOAD）

⚠️ Ask First（需确认再做）:
- 新增支持的文件类型（需更新白名单并评估存储/预览影响）
- 修改 S3 bucket 目录结构（影响现有文件路径，需数据迁移）
- 调整 CLIP 模型版本（需评估历史 embedding 兼容性）
- 修改文件大小上限（需评估 CDN 和 S3 带宽成本）

🚫 Never（禁止做）:
- 不得允许可执行文件上传（.exe / .sh / .bat / .js 等，服务端强制拒绝）
```

### 2.3 权限矩阵（Tab3 — tab-permission.html）— 对应 Issue #1880 #1887

**功能描述**: 超管在此配置素材的三档可见性规则（公开/团队/中标限定），编辑角色×操作维度的权限矩阵，并查看中标状态变更触发的权限刷新事件日志。

- **组件**:
  - `VisibilityRuleCards`（三档可见性规则说明卡：公开 / 团队成员 / 中标后全量，带可见范围示意图）
  - `RoleMatrixTable`（角色×操作维度的权限矩阵表格：行=角色，列=操作权限，单元格=开关 Switch）
  - `RoleAddModal`（新增角色弹窗）
  - `BidEventLog`（中标状态变更事件日志列表：时间 / 项目 / 变更类型 / 影响素材数 / 触发耗时）
  - `BidEventRefreshBtn`（手动触发刷新按钮，超管专用）
  - `MatrixExportBtn`（导出权限矩阵为 CSV）

- **数据源**:
  - `GET /wande/asset/permission/matrix`（获取全量 `role_permissions` 矩阵）
  - `PUT /wande/asset/permission/matrix`（批量保存矩阵变更，乐观锁保护）
  - `GET /wande/asset/bid-event/log`（分页查询中标事件日志，支持时间区间筛选）
  - `POST /wande/asset/bid-event/refresh`（手动触发全量权限刷新，超管权限）

- **交互**:
  - 矩阵表格单元格 Switch 切换 → 本地状态变更（未保存标记），点击「保存矩阵」批量提交
  - 新增角色 → 弹窗选择系统角色 → 加入矩阵行，初始权限全关
  - 中标事件日志实时滚动加载（IntersectionObserver 懒加载）
  - 手动触发刷新 → Loading 状态 → 后端异步执行全量权限同步 → 完成后 Toast 通知
  - 导出矩阵 → 生成 CSV 文件，文件名 `asset-permission-matrix-{YYYYMMDD}.csv`

- **代码现状**: 🟡 有基础需改造
  - `authService.java` 已有基础权限校验逻辑，但缺少素材维度的 `asset_project_relations` 关联查询

  - 需新建 `asset_project_relations` 表（详见 §4.3）
  - 权限矩阵 CRUD 需新建 `AssetPermissionController` + `AssetPermissionService`

- **已有文件**:
  - `backend/ruoyi-modules/wande-ai/src/main/java/org/ruoyi/wande/service/authService.java`

- **三级边界**:

```
✅ Always（必须做）:
- 权限变更必须写入 asset_audit_log（action=PERMISSION_CHANGE，detail 含 before/after diff）
- 中标事件触发权限刷新处理时延 ≤ 5s（Spring ApplicationEvent 同步监听，超时降级为异步补偿）
- 使用 Spring ApplicationEvent 解耦项目模块和素材模块（不得直接 Service 相互调用）
```

### 2.4 版本 & 日志（Tab4 — tab-version.html）— 对应 Issue #1793 #2163

- **组件**:
  - `AssetInfoCard`（当前素材基本信息：名称 / 类型 / 当前版本 / 最后更新时间）

  - `VersionUploadModal`（上传新版本弹窗：拖拽上传 + 变更说明文本框）
  - `VersionRollbackConfirm`（版本回退确认弹窗，显示回退影响提示）
  - `AuditLogTable`（审计日志表格：操作者 / 操作类型 / 素材 / 详情 / 时间 / IP，支持多维度筛选）
  - `WatermarkConfigPanel`（水印规则折叠面板：规则列表 CRUD + 条件编辑器 + 模板预览）

- **数据源**:
  - `GET /wande/asset/version/list?assetId=XX`（版本历史列表）
  - `POST /wande/asset/version/upload`（上传新版本，multipart，需附 `assetId` + `changelog`）
  - `POST /wande/asset/version/rollback`（回退到指定版本，body: `{assetId, targetVersionNo}`）
  - `GET /wande/asset/audit-log`（审计日志分页，筛选：`operator` / `action` / `timeRange` / `assetId`）

```
- 不得允许回退到不存在的版本号（版本号校验在 Service 层，不依赖前端传入的合法性）
```

### 2.5 AI 推荐（Tab5 — tab-ai-recommend.html）— 对应 Issue #1972

**功能描述**: 根据当前项目的属性（类型/行业/阶段/风格/适龄）智能推荐 TOP-30 素材，支持一键插入 PPT 方案，并通过用户反馈（标记不相关）持续优化推荐权重。

- **组件**:
  - `RecommendConditionPanel`（推荐条件设置面板：项目选择 / 项目类型 / 行业 / 阶段 / 风格 / 适龄）
  - `WeightExplainCard`（权重说明折叠卡：显示各维度权重比例 + 匹配逻辑说明）
  - `RecommendList`（推荐素材列表：卡片网格，带匹配度百分比 + 命中标签气泡 + 操作按钮）
  - `PPTInsertPanel`（PPT 一键插入面板：已选素材列表 + 页面位置选择 + 插入按钮）
  - `RecommendStats`（效果统计 4 卡：点击率 / 插入率 / 节省时间 / 素材覆盖率）
  - `FeedbackBtn`（单素材反馈按钮：「不相关」降权 / 「非常匹配」升权）

- **数据源**:
  - `POST /wande/asset/recommend`（推荐主接口，入参：`projectId` / `projectType` / `industry` / `stage` / `style` / `ageGroup`；出参：TOP-30 素材 + 匹配度 + 命中标签）
  - `POST /wande/asset/recommend/feedback`（反馈接口，入参：`recommendId` / `feedback`（-1/0/1）/ `reason`（可选））
  - `GET /wande/asset/recommend/stats`（效果统计，聚合：点击率 / 插入率 / 节省时间 / 覆盖率）
  - `POST /wande/ppt-plugin/insert`（PPT 插件调用，入参：`assetIds` / `slidePositions`）

- **交互**:
  - 进入 Tab 时自动读取当前项目上下文（若用户当前有活跃项目），预填推荐条件
  - 点击「立即推荐」→ Loading 状态（最长 2s）→ 返回 TOP-30 素材卡片列表
  - 素材卡片点击→加入候选列表（右侧 PPT 插入面板更新），再次点击取消
  - 「一键插入 PPT」→ 调用 PPT 插件 Webhook → 成功提示"已插入 N 张素材到方案第 X 页"
  - 「标记不相关」→ 调用 feedback 接口（feedback=-1）→ 该素材降权并从当前列表移除（不刷新全部列表）
  - 效果统计实时展示（每次推荐后更新），支持按时间区间筛选（7天/30天/自定义）

- **代码现状**: 🔴 需新建
  - `DesignAsset` / `DesignAssetTag` / `DesignAssetRecommendation` 三表均不存在（与 `assets` / `asset_recommendations` 整合，按 §4 方案统一设计）
  - pgvector 向量检索：评估 MySQL 8.0 D79 实例是否支持 `VECTOR` 类型；若不支持则使用 Milvus 独立部署，`asset_embedding` 表仅存 asset_id + model_version，向量存 Milvus
  - CLIP 微服务对接：新建 `ClipEmbeddingClient`（HTTP RestTemplate），异步生成 embedding 并存储
  - 推荐算法：向量余弦相似度（主权重 60%）+ 标签匹配（30%）+ 下载热度（10%），权重可配置化

- **三级边界**:

```
✅ Always（必须做）:
- 推荐接口响应时间 ≤ 2s（向量检索 + 权限过滤 + 排序），超时降级返回基于标签的精确匹配结果
- 推荐结果必须经过权限过滤（不得推荐用户无权访问的素材，过滤在 Service 层）
- 记录所有推荐事件（推荐条件 + 返回 assetId 列表）+ 点击事件 + 插入事件到 asset_recommendations
- 降权反馈（feedback=-1）实时更新权重（本次会话内立即生效，下次推荐时持久化）
- embedding 生成失败不阻塞素材上传，采用异步补偿（上传成功后放入补偿队列）

⚠️ Ask First（需确认再做）:
- 更换 embedding 模型版本（历史向量需批量重新生成，需评估计算成本和停机窗口）
- 调整各维度权重比例（需与设计部门和销售团队评审，避免"越推越同质"问题）
- 扩展 PPT 插件支持范围到 WPS Presentation / Keynote（需评估插件 API 差异和测试工作量）
- 向量数据库从 Milvus 迁移到 MySQL VECTOR 类型（需数据迁移脚本）

🚫 Never（禁止做）:
- 不得推荐超出用户权限范围的素材（Service 层双重校验：推荐前 + 返回前）
- 不得在客户端（浏览器/PPT 插件）缓存或传输向量（向量计算全部在服务端）
- 不得将 feedback 权重更新直接写死在推荐算法（权重计算抽象为可配置策略，方便 A/B 测试）
```

### 4.1 assets（素材主表）

**对应 Flyway**: `V20260425_01__create_assets_table.sql`

```sql
CREATE TABLE `assets` (
  `id`              BIGINT        NOT NULL AUTO_INCREMENT COMMENT '素材ID，主键',
  `category_id`     BIGINT        NOT NULL DEFAULT 0 COMMENT '分类节点ID，关联 asset_category.id',
  `asset_type`      VARCHAR(20)   NOT NULL COMMENT '资产类型：IMAGE/VIDEO/PDF/3D_MODEL/AUDIO',
  `file_type`       VARCHAR(20)   NOT NULL COMMENT '文件格式：jpg/png/mp4/pdf/psd/glb 等',
  `venue_type`      VARCHAR(30)   DEFAULT NULL COMMENT '场地类型：INDOOR/OUTDOOR/WATER_PARK/THEME_PARK',
  `age_group`       JSON          DEFAULT NULL COMMENT '适龄群体数组，如 ["0-3","4-12"]',
  `color_style`     JSON          DEFAULT NULL COMMENT '色彩风格数组，如 ["warm","neon","pastel"]',
  `material_type`   JSON          DEFAULT NULL COMMENT '材质类型数组，如 ["digital","3d_render"]',
  `name`            VARCHAR(200)  NOT NULL COMMENT '素材名称',
  `description`     TEXT          DEFAULT NULL COMMENT '素材描述',
  `s3_key`          VARCHAR(500)  NOT NULL COMMENT 'S3 对象路径（加密后），原始 URL 不暴露客户端',
  `thumb_s3_key`    VARCHAR(500)  DEFAULT NULL COMMENT 'S3 缩略图路径',
  `file_size`       BIGINT        NOT NULL DEFAULT 0 COMMENT '文件大小（字节）',
  `mime`            VARCHAR(100)  NOT NULL COMMENT 'MIME 类型，服务端探测，不信任客户端',
  `current_version` SMALLINT      NOT NULL DEFAULT 1 COMMENT '当前版本号',
  `uploader_id`     BIGINT        NOT NULL COMMENT '上传者 userId',
  `download_count`  INT           NOT NULL DEFAULT 0 COMMENT '下载次数（冗余计数，定时同步）',
  `status`          TINYINT       NOT NULL DEFAULT 1 COMMENT '状态：0=下架/1=正常',
  `tags`            JSON          DEFAULT NULL COMMENT 'AI 生成 + 手工编辑的标签数组',
  `create_time`     DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_time`     DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '最后更新时间',
  PRIMARY KEY (`id`),
  INDEX `idx_category_id` (`category_id`),
  INDEX `idx_asset_type` (`asset_type`),
  INDEX `idx_uploader_id` (`uploader_id`),
  INDEX `idx_status_create_time` (`status`, `create_time`),
  FULLTEXT INDEX `ft_name_desc_tags` (`name`, `description`) WITH PARSER ngram
  -- 注：tags 字段为 JSON，全文检索通过生成列实现，详见 Flyway 脚本
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='素材主表';
```

**索引说明**:
- `ft_name_desc_tags`：MySQL 8.0 ngram 全文索引，支持中文分词，用于 `MATCH AGAINST IN BOOLEAN MODE`
- `idx_status_create_time`：覆盖列表查询常用过滤条件

## 6. H5 适配方案

- **路由**: `/h5/asset`
- **组件库**: Vant 4（与 PC 端 Ant Design Vue 并存，按路由区分加载）
- **适配范围**: 仅 Tab1（素材浏览）和 Tab5（AI 推荐），Tab2/3/4 不提供 H5 版本

### 6.1 Tab1 H5 素材浏览

- 布局：顶部固定搜索栏 + 全屏卡片列表（单列）
- 组件替换：`ATree` → Vant `Sidebar` / `TreeSelect`；`APagination` → 下拉加载更多（IntersectionObserver）
- 卡片简化：仅展示缩略图 + 名称 + 类型标签，hover 操作浮层改为长按菜单（`van-action-sheet`）
- 搜索栏：Vant `Search` 组件，支持扫码（摄像头）触发图片相似搜索（预留，当前版本不实现）
- 筛选：底部弹出 `van-popup` 筛选面板，替代 PC 端顶部 FilterBar

### 6.2 Tab5 H5 AI 推荐

- 布局：顶部条件卡 + 下方推荐列表（单列）+ 底部固定「插入 PPT」按钮
- 条件面板简化：隐藏权重说明卡，条件项改为 Vant `Picker` / `Checkbox` 组合
- PPT 插入：H5 环境调用 PPT 插件 Webhook 与 PC 端相同，区别仅在 UI 交互
- 效果统计：H5 不展示（PC 专属超管视角）

### 6.3 H5 不适配的 Tab 说明

| Tab | 不适配原因 |
|-----|-----------|
| Tab2 上传分类 | 移动端文件上传限制大、AI 分类编辑操作复杂，H5 体验差；设计师上传场景在 PC 完成 |
| Tab3 权限矩阵 | 矩阵表格在移动端无法正常展示；权限管理属超管操作，非移动端场景 |
| Tab4 版本&日志 | 版本对比和审计日志需要大屏幕展示；版本上传操作在 PC 完成 |

## 7. PageGuide 内容

- **这是什么**: 万德方案素材统一数字资产管理中心（DAM），集中管理所有方案设计素材、图片、视频和 3D 模型。

- **解决什么问题**: 替代散落在本地硬盘/企业网盘/项目文件夹的素材管理方式。核心痛点：设计师在客户提出"要动物造型方案"时，平均耗时 20 分钟翻找素材；销售无法快速获取符合项目权限的素材；中标项目素材有泄露风险。

- **快速上手**（3 步）:
  1. **找素材**：进入「素材浏览」Tab → 左侧分类树选择主题 → 顶部多维度筛选（场地/年龄/色彩）→ 卡片 hover 下载（自动叠加水印）
  2. **上传素材**：进入「上传分类」Tab → 拖拽文件 → 等待 AI 自动识别分类和标签 → 绑定项目 → 批量保存
  3. **AI 推荐**：进入「AI 推荐」Tab → 选择项目和设计风格 → 点击「立即推荐」→ 从 TOP-30 结果中勾选素材 → 一键插入 PPT

- **权限说明**: 素材可见性与项目中标状态自动联动，中标后自动扩大全员访问权限，无需手动维护 ACL 清单。

## 附录 B：技术决策记录（ADR）

### ADR-001：向量存储选型

- **背景**: AI 推荐需要对 512 维向量进行余弦相似度检索，MySQL 8.0 D79 实例是否支持 `VECTOR` 类型待确认。
- **选项 A**: MySQL VECTOR 类型（8.0.36+ 支持），无新增基础设施
- **选项 B**: Milvus 独立部署，`asset_embedding_meta` 表存元数据，向量存 Milvus
- **决策**: 默认选方案 B（Milvus），理由：D79 实例版本不确定，Milvus 已有团队使用经验，检索性能更优
- **负责人**: 后端 CC，在 D80 之前确认并执行 Flyway V20260425_08

### ADR-002：水印实现方案

- **背景**: 需对图片和 PDF 两类文件格式叠加水印，且水印不可由普通用户关闭。
- **选项 A**: 服务端实时渲染水印（BufferedImage/iText），下载时动态生成
- **选项 B**: 预生成水印版本存储在 S3，按用户姓名/时间静态组合
- **决策**: 选方案 A（服务端实时渲染），理由：方案 B 存储成本高且无法包含动态的下载时间信息
- **实现要点**: 图片 → `BufferedImage`（JDK 内置，无额外依赖）；PDF → `iText 7 Community`（AGPL 许可）

### ADR-003：Spring ApplicationEvent vs MQ

- **背景**: 中标状态变更触发权限刷新，需要解耦项目模块和素材模块。
- **选项 A**: Spring ApplicationEvent（同进程内，无额外中间件）
- **选项 B**: RabbitMQ/Kafka（跨进程，需新增基础设施）
- **决策**: 选方案 A，Sprint-3 阶段保持简单，若后续两个服务拆分到不同微服务实例再迁移到 MQ
- **超时保护**: 监听器执行超过 3s 降级为异步补偿任务（`@Async` + `ThreadPoolTaskExecutor`），保证中标事件不阻塞主流程
