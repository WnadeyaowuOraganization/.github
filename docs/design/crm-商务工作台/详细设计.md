## 1. 页面概述

- **路由**: `/wande/crm/workbench`
- **菜单层级**: CRM > 商务工作台
- **权限标识**: `crm:workbench:view`（商务角色默认可见）
- **所属模块**: `module:fullstack`（前后端联动）
- **定位**: 项目矿场分配项目后，商务人员日常使用的CRM主界面。替代明道云"驾驶舱+日常过会管理+销售数据"三合一。

### 核心设计原则（来自最佳实践研究）

| 原则 | 说明 | 解决的明道云痛点 |
|------|------|---------------|
| 一次录入，全局继承 | 选商机=自动带出客户/负责人/部门/等级 | 8个表单重复填写客户名 |
| 流程驱动状态推进 | 下设计单→商机自动推进到设计阶段 | 手动改状态 |
| 透明度 > 惩罚 | 商务比管理层先看到异常 | 管理层过会才发现超期 |
| 自动记录替代手写日报 | 系统自动汇总操作，一键确认 | 日报大部分空白 |
| 异常驱动过会 | 只有异常/需决策项上议程 | 2606个全部过会 |
| 正向激励替代罚单 | 业绩+提成实时可见，KPI自动统计 | 罚单机制（当前0条） |

## 2. 页面布局（12个功能区域）

### 2.1 异常Banner — 超期商机醒目提示
- **组件**: 自定义AlertBanner（固定页面顶部，可关闭）
- **数据源**: `GET /api/crm/workbench/alerts`
- **逻辑**: 查询当前用户负责的商机中，最后活动时间 > 14天的记录数
- **交互**: 点击"查看详情"跳转到商机列表页，自动筛选超期商机
- **显示规则**: 当天首次打开显示，关闭后当天不再显示（localStorage控制）

### 2.2 状态卡片栏 — 5个KPI卡片
- **组件**: AStatistic × 5
- **数据源**: `GET /api/crm/workbench/stats`
- **字段**:

| 卡片 | 字段 | 颜色 | 点击行为 |
|------|------|------|---------|
| 需要关注的商机（异常） | `abnormal_count` | 红 | 跳转异常商机列表 |
| 本周待更新 | `pending_update_count` | 黄 | 跳转待更新列表 |
| 本月新商机 | `new_this_month` | 绿 | 跳转本月新建商机 |
| 有效商机总数 | `active_total` | 灰 | 跳转全部商机 |
| 今日自动汇总 | `today_auto_summary_status` | 蓝 | 滚动到今日汇总区域 |

### 2.3 商机管道（看板视图） — 核心操作区
- **组件**: 自定义KanbanBoard + ATable（双视图切换）
- **数据源**: `GET /api/crm/opportunities?owner=me&view=kanban`
- **看板列**: 6列，按商机阶段分组

| 阶段 | 字段值 | 描述 |
|------|--------|------|
| 新建商机 | `NEW` | 刚从矿场分配或手动创建 |
| 商务推进 | `PROGRESSING` | 正在跟进中 |
| 招投标 | `BIDDING` | 进入招投标阶段 |
| 签约 | `SIGNING` | 准备签约 |
| 回款 | `COLLECTING` | 已签约，跟踪回款 |
| 完结 | `CLOSED` | 项目完结 |

- **商机卡片字段**:
  - 标题（项目名称）
  - 客户名称（标注"自动继承"）
  - 预算金额
  - 阶段标签 + 异常标记（超14天无活动 → 红色边框+三角警告）
  - 最后活动时间（>14天红色，>7天黄色）
  - 操作按钮：[跟进记录] [下设计单]

- **列表视图字段**: 商机代码 | 项目名称 | 客户 | 阶段 | 预算 | 最后活动 | 状态 | 操作

### 2.4 待处理 — 分配给我的（v2.0新增）
- **组件**: 自定义TaskInbox
- **数据源**: `GET /api/crm/workbench/inbox`
- **4种来源**:

| 来源 | source_type | 颜色 | 操作 |
|------|-------------|------|------|
| 矿场分配 | `MINE_ASSIGN` | 红 | [接受] → 自动创建商机 + [查看] |
| 管理层催更新 | `NUDGE` | 黄 | [去更新] → 跳转商机详情 |
| 审批待办 | `APPROVAL` | 绿 | [去处理] → 跳转审批页 |
| 客户回复 | `CUSTOMER_REPLY` | 蓝 | [查看] → 跳转商机详情 |

- **红点逻辑**: 未处理的显示红点徽章，处理后消失
- **数量限制**: 最多显示最新10条，更多进"查看全部"

### 2.5 我的协同（等别人） — 跨部门协同追踪（v2.0新增）
- **组件**: 自定义CollabTracker
- **数据源**: `GET /api/crm/workbench/collaboration`
- **每条字段**: 事项类型 | 标题 | 当前处理人 | 等待天数

| 事项类型 | collab_type | 标签颜色 |
|---------|-------------|---------|
| 设计单 | `DESIGN_TASK` | 蓝 |
| 报价审批 | `QUOTE_APPROVAL` | 黄 |
| 合同审批 | `CONTRACT_APPROVAL` | 紫 |
| 投标审批 | `BID_APPROVAL` | 红 |

- **三种提醒按钮**:
  - 📢 Web平台提醒: `POST /api/notification/remind` → 对方Web端弹通知
  - 💬 企微提醒: `POST /api/wecom/message/send` → 企微消息推送
  - 📞 拨打电话: `tel:` 链接，从通讯录读取号码
- **等待天数**: ≥5天红色加粗，≥3天黄色，<3天灰色
- **底部链接**: "📒 公司通讯录 →" 跳转 `/wande/contacts`

### 2.6 今日活动自动汇总 — 替代手写日报
- **组件**: 自定义DailyAutoSummary
- **数据源**: `GET /api/crm/activity/today-summary`
- **逻辑**: 从 `activity_logs` 表查询当天该用户的所有自动+手动记录
- **每条字段**: 时间 | 活动描述 | 来源标签（已自动记录/待确认）
- **底部操作**: [确认今日汇总] 按钮 → `POST /api/crm/activity/confirm-daily`
- **确认后**: 该日摘要存入 `daily_summaries` 表，状态变为"已确认"

### 2.7 智能过会提醒 — 异常驱动
- **组件**: 自定义MeetingAgenda
- **数据源**: `GET /api/crm/meeting/agenda?type=smart`
- **筛选逻辑**:
  - 异常商机（>14天无活动）
  - 高概率成单待决策（成单预测≥80%）
  - 管理层标记"关注"的商机
- **每条**: 红/黄标签 + 商机名称 + 异常描述/决策说明
- **链接**: "查看全部过会商机(N)" → 跳转过会管理页

### 2.8 我的业绩 — 正向激励（v2.0新增）
- **组件**: 自定义PerformanceCard
- **数据源**: `GET /api/crm/performance/my`
- **字段**:

| 指标 | 字段 | 展示方式 |
|------|------|---------|
| 本月完成业绩 | `month_achievement` | 大数字 + 进度条 |
| 本月目标 | `month_target` | 进度条标注 |
| 月度完成率 | `month_rate` | 百分比+颜色 |
| 年度累计 | `year_achievement` | 大数字 + 进度条 |
| 年度目标 | `year_target` | 进度条标注 |
| 年度完成率 | `year_rate` | 百分比 |
| 预计提成 | `estimated_commission` | 金额 |

- **数据来源**: 合同签约金额（已回款部分）→ 提成规则自动计算
- **链接**: "查看提成明细 →" → 跳转 `/wande/hr/commission`

### 2.9 待办清单 — 今日任务
- **组件**: ATodoList（Ant Design Checkbox + 列表）
- **数据源**: `GET /api/crm/workbench/todos`
- **字段**: 任务描述 | 截止时间 | 关联商机
- **操作**: 勾选完成 → `PATCH /api/crm/todo/{id}/complete`

### 2.10 最近跟进记录 — 自动+手动混合
- **组件**: ATable
- **数据源**: `GET /api/crm/activity/recent?limit=5`
- **列**: 时间 | 商机 | 记录类型 | 内容摘要 | 来源（自动/手动标签）

### 2.11 快捷操作FAB — 右下角浮动按钮（v2.0新增）
- **组件**: 自定义QuickActionFab（fixed定位）
- **交互**: 点击"+"展开面板，再次点击收起，"+"旋转45°变"×"
- **6个快捷入口**:

| 入口 | 跳转/弹窗 | 简化表单字段 |
|------|---------|-----------|
| 新建商机 | 弹窗表单 | 项目名称、客户、预算 |
| 下设计单 | 弹窗表单 | 商机（自动带出客户/等级）、设计需求 |
| 提交报价 | 弹窗表单 | 商机（自动带出）、报价金额、附件 |
| 申请投标(HZ) | 弹窗表单 | 商机、招投标编号、开标日期 |
| 新建合同 | 弹窗表单 | 商机（自动带出客户）、合同金额、合同类型 |
| 写跟进记录 | 弹窗表单 | 商机、活动类型、结果、简要说明 |

- **核心原则**: 选商机后自动继承客户/负责人/部门/等级，商务只填3-5个核心字段

## 4. 数据库设计

### 已有表（复用）
- `opportunities` — 商机表（backend#41/#57/#150）
- `customers` — 客户主数据表（backend#3）
- `activity_logs` — 活动日志表（backend#1043）
- `contracts` — 合同表（backend#65）
- `payment_plans` — 回款计划表（backend#157）

### 新增表

#### `crm_inbox` — 商务收件箱
```sql
CREATE TABLE crm_inbox (
  id BIGSERIAL PRIMARY KEY,
  user_id BIGINT NOT NULL,           -- 被分配人
  source_type VARCHAR(32) NOT NULL,  -- MINE_ASSIGN/NUDGE/APPROVAL/CUSTOMER_REPLY
  title VARCHAR(512) NOT NULL,
  description TEXT,
  assigner_id BIGINT,                -- 分配人
  status VARCHAR(16) DEFAULT 'PENDING', -- PENDING/ACCEPTED/DISMISSED
  related_opportunity_id BIGINT,
  related_mine_project_id BIGINT,
  created_at TIMESTAMP DEFAULT NOW(),
  processed_at TIMESTAMP
);
```

#### `crm_collaboration` — 协同追踪
```sql
CREATE TABLE crm_collaboration (
  id BIGSERIAL PRIMARY KEY,
  initiator_id BIGINT NOT NULL,      -- 发起人（商务）
  collab_type VARCHAR(32) NOT NULL,  -- DESIGN_TASK/QUOTE_APPROVAL/CONTRACT_APPROVAL/BID_APPROVAL
  title VARCHAR(512) NOT NULL,
  current_handler_id BIGINT,         -- 当前在谁那里
  waiting_since TIMESTAMP,           -- 开始等待时间
  status VARCHAR(16) DEFAULT 'WAITING', -- WAITING/COMPLETED/CANCELLED
  related_opportunity_id BIGINT,
  created_at TIMESTAMP DEFAULT NOW()
);
```

#### `daily_summaries` — 每日汇总确认
```sql
CREATE TABLE daily_summaries (
  id BIGSERIAL PRIMARY KEY,
  user_id BIGINT NOT NULL,
  summary_date DATE NOT NULL,
  auto_activities_count INT DEFAULT 0,
  manual_activities_count INT DEFAULT 0,
  status VARCHAR(16) DEFAULT 'DRAFT', -- DRAFT/CONFIRMED
  confirmed_at TIMESTAMP,
  UNIQUE(user_id, summary_date)
);
```

#### `ownership_change_logs` — 责任人变更日志
```sql
CREATE TABLE ownership_change_logs (
  id BIGSERIAL PRIMARY KEY,
  entity_type VARCHAR(32) NOT NULL,   -- LEAD/OPPORTUNITY/CUSTOMER
  entity_id BIGINT NOT NULL,
  previous_owner_id BIGINT,
  new_owner_id BIGINT NOT NULL,
  change_reason TEXT,
  changed_by BIGINT NOT NULL,
  changed_at TIMESTAMP DEFAULT NOW()
);
```

## 7. PageGuide内容
