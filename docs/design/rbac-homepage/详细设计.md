## 1. 页面概述

- **路由**: `/wande-ops/dashboard`（登录后默认跳转）
- **菜单层级**: 一级入口（根路由）
- **权限标识**: 按角色动态渲染（见§2 RBAC矩阵）
- **所属模块**: `module:fullstack`（前端菜单渲染 + 后端角色权限API）

### 1.1 核心设计思路

当前3083侧边栏有18个一级菜单、100+子菜单，全部平铺展示。本方案将其按4个真实岗位（商务/支持中心/项目安装/综合管理）+ Boss角色进行RBAC分组，每个角色：
- 只看到与自己岗位相关的侧边栏菜单
- 登录后进入专属的Dashboard主页（角色专属Widget）
- 公用功能（项目中心、AI对话等）所有角色可见

## 4. API契约

### 4.1 角色菜单API

#### `GET /api/wande/menu/my`
根据当前用户角色返回可见菜单树。

**返回**:
```json
{
  "code": 200,
  "data": [
    {
      "menuId": 1,
      "menuName": "CRM客户管理",
      "icon": "users",
      "path": "/wande-ops/crm",
      "children": [
        { "menuId": 101, "menuName": "客户列表", "path": "/wande-ops/crm/customer" }
      ]
    }
  ]
}
```

#### `GET /api/wande/dashboard/widgets`
根据当前用户角色返回Dashboard Widget配置。

**返回**:
```json
{
  "code": 200,
  "data": {
    "role": "sales",
    "widgets": [
      { "type": "announcement", "title": "公司公告", "size": "1/3", "api": "/api/wande/announcement/latest?limit=3" },
      { "type": "stat-card", "title": "我的客户摘要", "size": "1/3", "api": "/api/wande/crm/stats/my" }
    ]
  }
}
```
