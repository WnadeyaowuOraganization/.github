### §2.1 Phase 1 · 营销序列引擎

> 对应 Issue: #2793（数据模型）/ #2798（模板引擎）/ #2800（冷外展 7 步）/ #2802（A/B 测试）

#### §2.1.1 序列管理 — `phase1/tab-sequences.html`（对应 #2793）
- **组件**: a-table + a-drawer + a-form
- **数据源**: `GET /outreach/sequences?status=&channel=&creator=&keyword=`
- **交互**: 筛选 → 表格分页；点击行 → 侧边抽屉展开数据模型 JSON + API 端点清单；[新建序列]按钮 → 表单向导
- **代码现状**: 🔴 需新建
- **三级边界**:
  - ✅ Always: 序列 CRUD 必须走审计日志（operation_log 表）
  - ⚠️ Ask First: 删除运行中序列（有活跃线索）需二次确认
  - 🚫 Never: 不得硬删除（仅软删 + 归档）

#### §2.1.2 模板编辑器 — `phase1/tab-templates.html`（对应 #2798）
- **组件**: 三栏布局（列表 + 编辑器 + 变量面板）
- **数据源**: `GET /outreach/templates` + `POST /outreach/templates/{id}/render`（变量替换预览）
- **交互**: 左列分组切换 → 中编辑器加载富文本 → 右变量拖入编辑区；[测试发送]按钮 → 发送到指定邮箱
- **代码现状**: 🔴 需新建
- **变量规范**: `{{first_name}}` / `{{company}}` / `{{project_city}}` / `{{industry}}` 等预置 15 个

#### §2.1.3 冷外展 7 步序列 — `phase1/tab-cold-outreach.html`（对应 #2800）
- **组件**: 时间轴 + 预置模板卡
- **数据源**: `GET /outreach/cold-outreach-templates`
- **预置模板**: 欧美 7 步 / 东南亚 7 步 / 中东 7 步（Day 0/3/7/14/21/30/45）
- **交互**: 展开模板 → 查看每步主题/摘要/时机；[克隆为我的序列]按钮 → 创建序列副本

#### §2.1.4 A/B 测试 — `phase1/tab-abtest.html`（对应 #2802）
- **组件**: 测试列表 + 详情卡（A/B 条形图）
- **数据源**: `GET /outreach/ab-tests` + `GET /outreach/ab-tests/{id}/stats`
- **统计**: 打开率/点击率/回复率，p < 0.05 自动标注显著性
- **交互**: 新建测试 → 选类型（主题/正文/时间）→ 分配流量比例

### §2.2 Phase 2 · 邮件外展

> 对应 Issue: #2796 / #2797 / #2801 / #2803 / #2804 / #2806

#### §2.2.1 SMTP 配置 — `phase2/tab-smtp.html`（对应 #2796）
- **组件**: 域名列表 + DNS 验证状态
- **数据源**: `GET /outreach/smtp/domains` + `POST /outreach/smtp/domains/{id}/verify-dns`
- **关键字段**: 域名 / SMTP 提供商 / DKIM / SPF / DMARC / 日发送限额 / 已用量
- **代码现状**: 🔴 需新建
- **三级边界**:
  - ✅ Always: 新增域名必须先验证 DNS 通过才允许发送
  - ⚠️ Ask First: 修改已启用域名的 SMTP 凭据需运维确认
  - 🚫 Never: 不得明文存储 SMTP 密码（加密存储）

#### §2.2.2 预热+送达率监控 — `phase2/tab-deliverability.html`（对应 #2797）
- **组件**: KPI 卡 + 曲线图占位 + 黑名单检测
- **数据源**: `GET /outreach/deliverability/warmup` + `GET /outreach/deliverability/blacklist-check`
- **预热规则**: Day 1 发 5 封 → Day 7 发 30 封 → Day 30 发 150 封 → Day 56 达到目标
- **黑名单检测**: Spamhaus / Barracuda / SpamCop（每 6 小时定时检查）

#### §2.2.3 邮件追踪 — `phase2/tab-tracking.html`（对应 #2801）
- **组件**: 事件流表格 + 追踪详情抽屉
- **数据源**: `GET /outreach/tracking/events`
- **追踪方式**: 追踪像素（1x1 gif）+ 短链点击 + IMAP 回复检测
- **事件类型**: 发送/送达/打开/点击/回复/退订/退回

#### §2.2.4 收件箱轮转 — `phase2/tab-rotation.html`（对应 #2803）
- **组件**: 发送池 + 轮转策略 + 实时配额
- **数据源**: `GET /outreach/rotation/pools` + `GET /outreach/rotation/quota-realtime`
- **策略**: round-robin / 加权 / 按回复率自适应
- **防垃圾规则**: 每小时最多 20 封 / 同域名间隔 10 分钟 / 夜间暂停

#### §2.2.5 外展分析看板 — `phase2/tab-analytics.html`（对应 #2804）
- **组件**: 5 KPI + 漏斗图 + 序列对比表 + 时间热力图
- **数据源**: `GET /outreach/analytics/summary?sequence=&range=`
- **漏斗**: 发送 → 送达 → 打开 → 点击 → 回复 → 会议

#### §2.2.6 回复智能分类 — `phase2/tab-reply-classify.html`（对应 #2806）
- **组件**: 待分类队列 + AI 分类结果 + 自动化规则
- **数据源**: `POST /outreach/reply-classify` (AI) + `POST /crm/opportunities/auto-create`
- **AI 分类**: 感兴趣 / 需信息 / 转介绍 / 不感兴趣 / 退订 / 自动回复（6 类）
- **自动化规则**: 感兴趣 → 自动创建商机 + 分配销售；不感兴趣 → 结束序列
- **三级边界**:
  - ✅ Always: AI 分类置信度 < 85% 需人工确认
  - ⚠️ Ask First: 新增分类类别需更新训练数据
  - 🚫 Never: 不得自动回复客户（只分类，人工触达）

## §10 关联业务域

- **输入**: 海关/展会/LinkedIn/企微扫码/SEM 表单 → 原始线索池
- **本模块**: 外展序列触达 + 多渠道信号融合 + 评分 + 移交
- **输出**: 合格商机自动流向 CRM 商务中心（#37 biz:crm）/ 项目矿场（#7）

### 数据流
```
Phase 0 采集 → 原始 Lead 池
  → Phase 1 序列（邮件/企微/LinkedIn）
  → Phase 2/3/4 执行触达 + 追踪信号
  → Phase 5.1 评分融合
  → Phase 5.3 移交决策
  → CRM 商机 + 分配销售
  → Phase 5.2 ROI 归因
```
