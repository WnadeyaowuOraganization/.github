## 1. 模块概述

### 1.1 模块定位

**中标后执行管理** — 从中标签约到售后质保的全生命周期项目执行管理模块。

支持三种业务模式，各自独立的阶段流程：

| 业务模式 | 阶段数 | 适用场景 |
|---------|--------|---------|
| 直销（Direct） | 12 阶段 | 直接面向终端客户的销售项目 |
| 经销（Dealer） | 8 阶段 | 通过经销商渠道的销售项目 |
| 国贸（International） | 10 阶段 | 涉及国际贸易的出口项目 |

### 1.2 路由规划

| 页面 | 路由 | 说明 |
|------|------|------|
| 项目列表页 | `/wande/project/execution` | 所有执行中项目的汇总视图 |
| 项目详情页 | `/wande/project/execution/:id` | 单个项目的全生命周期详情 |

- **菜单层级**: 项目中心 > 执行管理
- **权限标识前缀**: `project:execution:`
- **所属模块**: `module:fullstack`

### 1.3 原型说明

原型已部署，共 10 个 HTML 文件：

- 项目列表页（1个）
- 项目详情页 8 个功能域 Tab（8个）
- 其他辅助页面（1个）

**4 个统计卡片**：

| 卡片 | 字段 | 说明 |
|------|------|------|
| 在执行项目数 | `activeCount` | 含直销/经销/国贸分别计数 |

- **组件**: `a-radio-group`（业务模式）+ `a-select`（执行阶段）+ `a-tag`（项目状态）+ `a-input-search`
- **交互**: 任意筛选条件变化均触发列表刷新（防抖 300ms）

**筛选项定义**：

| 筛选项 | 组件 | 取值 | 说明 |

- **组件**: 自定义组件 `ExecutionTimeline`（基于 `a-timeline` 扩展）
- **数据源**: `GET /api/project/execution/{id}/stages`（含子任务）
- **代码现状**: 🔴 需新建

**每个阶段节点包含**：

| 元素 | 说明 |
|------|------|
| 状态指示器 | 已完成/进行中/未开始/延期 |
| 阶段名 | 含阶段序号 |
| 计划时间 / 实际时间 | 双日期展示，延期标红 |
| 负责人 | 头像 + 姓名 |
| 关键产出物 | 文件/里程碑标记 |
| 备注 | 折叠展示 |

**交互**：点击当前阶段展开子任务详情

**业务规则**：

✅ **Always**：
- 所有日期使用 `LocalDate`，不带时区
- 阶段推进必须走审批引擎（D66）确认，不可直接跳阶段
- 阶段回退必须有审批记录

⚠️ **Ask First**：
- 子任务是否需要自动从工艺路线模板生成

🚫 **Never**：
- 不要允许跨阶段跳转（必须顺序推进，除非审批通过的特殊回退）
- 不要硬编码阶段顺序，应从配置读取

## 3. API 契约

> 所有接口均使用 `application/json`，鉴权通过 JWT Token（Header: `Authorization: Bearer <token>`）。返回格式统一为 `R<T>` 包装对象，分页列表返回 `TableDataInfo<T>`。

  byMode: {
    direct: int,
    dealer: int,
