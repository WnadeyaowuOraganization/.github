## 1. 页面概述

- **路由**:
  - `/shared/approval` — 全员审批中心（待办/已办/发起/抄送）
  - `/admin/approval/templates` — 管理员：表单模板管理
  - `/admin/approval/simplification` — 管理员：流程简化配置
  - `/admin/approval/analytics` — 管理员：效率分析看板
- **菜单层级**:
  - 公共板块 > 审批中心（全员可见）
  - 超管驾驶舱 > 审批管理 > 模板管理 / 流程简化 / 效率分析（管理员专用）
- **权限标识**:
  - `approval:center:view` — 查看审批中心
  - `approval:center:approve` — 审批操作（通过/驳回/退回/转交/加签）
  - `approval:center:initiate` — 发起审批
  - `approval:admin:template` — 模板管理
  - `approval:admin:rule` — 规则配置（跳过/自动审批/路由）
  - `approval:admin:analytics` — 效率分析看板
- **所属模块**: `module:fullstack`
- **底层引擎**: ruoyi-ai 内置 Flowable 工作流引擎（Java后端 + 前端组件直接复用）
- **企微贯通**: 审批消息推送至企微，H5端处理（`views/h5/approval/index.vue`）

- `GET /api/approval/cc?page=&size=` — 抄送我的列表

- `POST /api/approval/{taskId}/reject` — 驳回

**交互细节**:
- 超时判定: 等待时长（当前时间 - 节点进入时间）> 模板配置的SLA阈值小时数
- 批量通过：选中审批项后按钮激活，点击后弹出确认框，成功后刷新列表
- 撤回确认弹窗: 「确认撤回后，流程终止，无法恢复」
- 键盘快捷键: ESC 关闭弹窗

#### 三级边界规则（#2366）

```
✅ Always（必须做）:
- 复用上游 approval-modal.vue 和 approval-timeline.vue 作为详情弹窗基础
- 复用 ApprovalQueue 实体和 ApprovalQueueServiceImpl 服务层
- 超时标记必须基于模板SLA配置，不能硬编码小时数
- 批量通过需要后端幂等处理（重复提交安全）
- 审批意见字段对通过操作为选填，对驳回操作为必填

⚠️ Ask First（需确认再做）:
- SLA阈值是否按模板独立配置，还是全局统一配置
- 驳回时是否支持"退回到指定节点"（非仅退回到上一节点）
- 加签操作：会签模式还是或签模式
- 待办列表默认排序规则（超时优先 or 时间倒序）

🚫 Never（禁止做）:
- 不允许审批人审批自己发起的流程（需后端校验）
- 已通过/已驳回的审批单禁止撤回
- 不允许越权查看他人发起的审批详情（非抄送/非管理员）
- 不得在前端硬编码审批节点名称
```

**页面结构**:

```
审批进度追踪
├── 顶部选择栏（审批实例下拉 + 视图切换：流程图/时间轴）
│
├── [流程图视图]
│   ├── DAG 流程图渲染区（ApprovalProgressTracker 组件）
│   │   ├── 节点渲染规则:
│   │   │   ├── 已完成: 绿色背景 #22c55e
│   │   │   ├── 当前处理中: 蓝色背景 #3b82f6 + pulse动画
│   │   │   ├── 未到达: 灰色 #e5e7eb
│   │   │   └── 驳回: 红色 #ef4444
│   │   ├── 布局模式: 串行（水平箭头连线）/ 并行（分叉汇合）/ 会签（多审批人并排）
│   │   └── 节点点击 → NodeDetailPopover 弹出（处理人/耗时/意见）
│   ├── 统计卡片（总节点数 / 已完成 / 当前节点名 / 预估剩余时间）
│   └── 卡点预警区（当前节点等待时长 + SLA使用率百分比）
│
└── [时间轴视图]
    ├── 纵向时间线（上→下，已发生在前）
    │   └── 每节点: 状态圆点 + 头像 + 处理人 + 操作结果 + 耗时 + 时间戳 + 审批意见引用框
    └── 底部预估剩余时间（基于历史同类审批平均耗时）
```

**DAG组件技术方案**:
- 使用 `@antv/g6` 或 `vue-flow`（@vue-flow/core）渲染DAG
- 节点数据由后端API返回，格式见 API 契约
- 流程图支持水平滚动（overflow-x: auto）适配多节点场景
- 移动端降级为时间轴视图（宽度 < 768px 自动切换）

**数据源 API**:
- `GET /api/approval/instances?status=all&page=&size=` — 审批实例下拉列表
- `GET /api/approval/instance/{instanceId}/progress` — 实例进度详情
- `GET /api/approval/instance/{instanceId}/estimate` — 预估剩余时间

**预估算法说明**:
- 基于历史同类模板的各节点平均耗时计算
- 剩余节点平均耗时之和 = 预估剩余时间
- 数据存储在效率分析聚合表，每日凌晨计算刷新

#### 三级边界规则（#3149, #3150）

```
✅ Always（必须做）:
- 流程图节点状态实时轮询（每60秒自动刷新，或WebSocket推送）
- 当前节点必须有呼吸动画区别于其他节点
- 串行/并行/会签三种布局都必须支持
- 移动端自动降级到时间轴视图

⚠️ Ask First（需确认再做）:
- DAG库选型：@antv/g6 vs vue-flow（前者功能更强但包体大，后者轻量）
- 流程图是否需要缩放/拖拽功能（用于节点多的复杂流程）
- 是否支持"催办"操作入口放在进度追踪页面

🚫 Never（禁止做）:
- 不允许直接渲染 Flowable 内部 BPMN XML，需后端转换为统一数据结构
- 不允许在客户端计算预估时间（必须走后端接口）
- 流程图节点不允许展示其他审批人的联系方式（隐私保护）
```

**条件显隐引擎（condition-evaluator.ts）**:

```typescript
// 支持的操作符
type Operator = '==' | '!=' | '>' | '>=' | '<' | '<=' | 'in' | 'not_in' | 'empty' | 'not_empty';

interface ConditionRule {
  field: string;     // 依赖字段的key
```

```
- 是否需要报表导出功能（Excel/PDF）
- 效率数据是否按部门隔离（普通管理员只看本部门数据）

🚫 Never（禁止做）:
- 不允许展示具体审批人的个人绩效数据（如张三平均处理速度）
- 分析数据不允许实时查询原始表（必须走聚合表）
```

## 6. H5 适配方案

**已有文件**: `views/h5/approval/index.vue` + `api/wande/approval-h5.ts`

**企微H5场景**: 审批人在企微消息中收到审批通知 → 点击跳转H5页面 → 快速通过/驳回

**H5页面功能范围**（精简版，不是完整PC功能）:

```
H5审批页（/h5/approval?taskId=xxx&token=xxx）
├── 顶部: 标题 + 发起人信息
├── 业务数据区: 表单数据只读展示（FormRenderer的readOnly模式）
├── 审批进度区: 简化时间轴（3-5节点）
├── 意见输入框: textarea（驳回时必填）
└── 底部操作栏（固定到屏幕底部）:
    ├── 通过按钮（绿色，全宽）
    └── 驳回按钮（红色，outline）
```

**H5特殊处理**:
- token通过URL参数传递（企微免登录凭证）
- FormRenderer在H5下强制进入readOnly模式（H5不支持发起审批）
- 热力图、AI建议、模板管理等功能不在H5端提供
- 宽度适配: viewport meta + rem布局，最小宽度320px
- 文件附件: 仅支持查看（调用企微内置文件预览）

**需改造内容**（相对现有 `views/h5/approval/index.vue`）:
- 接入 FormRenderer 组件（readOnly模式），替换现有的静态HTML数据展示
- 接入审批进度时间线（TimelineView轻量版）
