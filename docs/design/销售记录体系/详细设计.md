## 1. 页面概述

### 1.1 销售记录中心（多Tab）
- **路由**: `/wande/ops-hub/record-center`
- **菜单层级**: 业务运营中心 > 销售记录中心
- **权限标识**: `ops_hub:record_center:view`
- **所属模块**: `module:ops-hub` + `module:fullstack`
- **架构**: 多Tab架构（4Tab），index.html + tab-timeline/tab-grouped/tab-weekly/tab-nudge

### 1.2 老板周报（单页）
- **路由**: `/wande/ops-hub/boss-report`
- **菜单层级**: 业务运营中心 > 老板周报
- **权限标识**: `ops_hub:boss_report:view`
- **所属模块**: `module:ops-hub` + `module:backend`（生成引擎）+ `module:frontend`（预览页）
- **触发**: 每周五 16:00（北京时间）定时任务自动生成 + 企微推送

## 2. 页面布局 — 销售记录中心

### 2.1 Tab：时间线视角

- **原型文件**: `prototype/tab-timeline.html`
- **组件**:
  - 顶部 `<a-select mode="multiple">` 记录类型筛选 + `<a-select>` 商务/项目/客户 + `<a-range-picker>` 时间
  - 主体自研 `<TimelineList>`（按日期分组，每条记录卡片含类型徽章+左侧色条+操作行）
  - 右侧 `<a-drawer>` 详情侧滑
- **数据源**:
  - `GET /wande/ops-hub/records/timeline?types=&salesIds=&projectIds=&customerIds=&timeRange=&page=&size=`
  - `GET /wande/ops-hub/records/{recordId}/detail`
- **交互**:
  - 7类型多选筛选、商务/项目/客户筛选、时间范围切换 → 列表实时刷新
  - 点击「查看详情」→ 抽屉滑出，含完整内容+关联实体+评论批注+Nudge状态
  - 行内操作：📌标记重要 / ✏️添加批注 / 📞催更新Nudge / 🔗跳详情
- **代码现状**: 🟡 有基础需改造
  - 已有 `wdpp_activity_logs` 表（来自 #1459），但需扩展 customer_id 字段+多实体关联表
  - 已有 `RecordCenterController` 雏形（#2211），但视角切换逻辑不完整
- **三级边界**:
  - ✅ Always: 默认加载本周数据；分页每页20条；按时间倒序
  - ⚠️ Ask First: 抽屉中是否允许编辑评论（目前只允许新增）
  - 🚫 Never: 不在前端聚合多实体关联（后端预聚合返回）

### 2.2 Tab：三视角分组

- **原型文件**: `prototype/tab-grouped.html`
- **组件**:
  - 顶部分组方式按钮组（按商务/按项目/按客户）+ 时间范围 + 排序
  - 视图A：商务卡片网格（3列），含KPI区+折叠展开
  - 视图B：项目列表（行展开为迷你时间线）
  - 视图C：客户卡片（下挂多项目）
- **数据源**:
  - `GET /wande/ops-hub/records/grouped?dimension=sales|project|customer&timeRange=&sortBy=`
- **交互**:
  - 分组方式切换 → 整页切换，3视图共用一个数据源不同聚合
  - 卡片/项目展开 → 加载该实体下的记录列表
  - 异常项目（>7天未跟进）行/卡片标橙
  - 异常商务（本周零活动）卡片虚线边框+⚠️角标
- **代码现状**: 🔴 需新建（原 #1262/#2211 仅做了时间线视角）
- **三级边界**:
  - ✅ Always: 三视图共用一个 dimension 参数，后端按维度聚合
  - ⚠️ Ask First: 客户视角下「全部项目」如何排序（按金额/按活跃度）
  - 🚫 Never: 不前端做聚合切换，每次切换调后端

### 2.3 Tab：项目周报

- **原型文件**: `prototype/tab-weekly.html`
- **组件**:
  - 顶部周次选择器 ← W17 (本周) → + 导出按钮组
  - 4 KPI 卡片
  - 主体：商务-项目两级折叠卡片（`<details>` 原生）
- **数据源**:
  - `GET /wande/ops-hub/weekly-report?week=2026-W17`
  - `POST /wande/ops-hub/weekly-report/export?format=pdf|excel&week=`
  - `POST /wande/ops-hub/weekly-report/batch-nudge` (批量催更新)
- **交互**:
  - 周次切换 → 整页数据刷新
  - 商务卡片展开 → 显示该商务负责的项目列表
  - 项目展开 → 该项目本周活动迷你时间线
  - 异常高亮：本周零活动商务⚠️ / >7天未跟进项目🟠
  - 行内 Nudge 按钮 + 顶部批量 Nudge
- **代码现状**: 🟡 有基础需改造
  - 已有 `WeeklyReportService`（#2214）能聚合本周数据，但缺商务-项目两级展开+异常高亮
  - 前端页面 `weekly-report/index.vue`（#1259）已存在，需重写布局对齐原型
- **三级边界**:
  - ✅ Always: 周次按 ISO Week（周一为第一天）；默认本周
  - ⚠️ Ask First: 导出 PDF 模板需先提供样例
  - 🚫 Never: 不让普通商务编辑别人的周报

### 2.4 Tab：Nudge管理

- **原型文件**: `prototype/tab-nudge.html`
- **组件**:
  - 顶部异常告警条（>24h未响应）+ KPI卡片（4个）+ 30天趋势小图（CSS+SVG）
  - 4状态多选筛选 + 发起人/被催商务下拉 + 项目搜索 + 时间范围
  - 主体表格 + 行点击抽屉
- **数据源**:
  - `GET /wande/ops-hub/nudges?statuses=&initiators=&targets=&projects=&timeRange=&page=&size=`
  - `GET /wande/ops-hub/nudges/{nudgeId}/context` (上下文：关联项目+客户+前后活动时间线)
  - `POST /wande/ops-hub/nudges/{nudgeId}/escalate` (升级给上级)
  - `POST /wande/ops-hub/nudges/{nudgeId}/remind` (重新提醒)
  - `GET /wande/ops-hub/nudges/trend?days=30` (趋势数据)
- **交互**:
  - 4状态筛选、超时未响应一键升级
  - 行点击 → 详情抽屉显示完整上下文
  - 顶部告警条「一键全部升级」批量操作
- **代码现状**: 🟡 有基础需改造
  - 已有 Nudge API（#1261/#2212）但缺管理视角（吴耀目前看不到所有Nudge的全局列表）
  - 缺超时24h自动升级逻辑+趋势统计
- **三级边界**:
  - ✅ Always: 24h未响应自动标记超时（后端定时扫描）
  - ⚠️ Ask First: 升级给上级的规则（吴耀的上级是谁？或直接发邮件给副总）
  - 🚫 Never: 不允许普通商务催别人

## 3. 页面布局 — 老板周报

### 3.1 Section：本周概要
- **原型文件**: `prototype-boss-report/prototype.html`
- **组件**: 4 KPI 大卡片 + 一句话 AI 总结
- **数据源**: `GET /wande/ops-hub/boss-report?week=2026-W17`
- **代码现状**: 🟡 有基础需改造（#958/#1460 已实现 BossWeeklyReportService 雏形，需对齐新结构）

### 3.2 Section：异常预警
- **组件**: 4类警告卡（零活动商务/超期项目/Nudge未响应/新增丢标），每条带「立即处理」CTA
- **数据源**: 同 §3.1（`exceptions[]` 字段）
- **交互**: 点击CTA → 跳转对应处理页（催更新/查看项目/复盘）

### 3.3 Section：各商务一句话摘要
- **组件**: 8张商务卡片网格
- **数据源**: 同 §3.1（`salesSummaries[]` 字段）
- **AI 摘要**: 调用本地 Qwen-7B 模型对商务本周活动做一句话归纳

### 3.4 Section：管道健康度
- **组件**:
  - 圆环仪表盘（conic-gradient）显示管道覆盖率
  - 5阶段堆叠条形图（早期/投资/谈判/确认/签约）
  - 加权管道值 + 阶段流转健康度
- **数据源**: 同 §3.1（`pipelineHealth` 字段）
- **代码现状**: 🟡 已有 SalesKpiView (#1460) 提供基础数据

### 3.5 Section：推荐行动（AI建议）
- **组件**: 3-5条AI建议卡片，每条含 标题+理由+建议动作+采纳按钮
- **数据源**: `POST /wande/ops-hub/boss-report/ai-suggestion` (调用本地模型)
- **代码现状**: 🔴 需新建（依赖 wande-org-update 组织技能 §SOP-本地模型）

### 3.6 周次选择 + 推送状态 + 导出
- **组件**: 顶部周次选择器（含历史下拉）+ 推送状态标识 + 重发企微/导出PDF/打印按钮
- **数据源**:
  - `POST /wande/ops-hub/boss-report/repush?week=` 重发企微
  - `POST /wande/ops-hub/boss-report/export?week=&format=pdf` 导出
- **历史查看**: 周次下拉显示最近12周存档

```
SALES_LOG       销售记录（商务手动写）
WEEKLY_REPORT   周报
MONTHLY_REPORT  月报
SYSTEM_AUTO     系统自动（流程节点触发）

```

### 5.2 新建 wdpp_record_entity_links 表（多对多）
```sql
CREATE TABLE wdpp_record_entity_links (
  id BIGINT AUTO_INCREMENT PRIMARY KEY,
  record_id BIGINT NOT NULL COMMENT '记录ID（关联 wdpp_activity_logs）',
  entity_type VARCHAR(20) NOT NULL COMMENT 'PROJECT|CUSTOMER|SALES',
  entity_id BIGINT NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_record (record_id),
  INDEX idx_entity (entity_type, entity_id)
```

- Flyway: `V20260421_002__create_record_entity_links.sql`

### 5.3 新建 wdpp_nudges 表（如不存在）
```sql
CREATE TABLE wdpp_nudges (
  id BIGINT AUTO_INCREMENT PRIMARY KEY,
  initiator_id BIGINT NOT NULL,
  target_user_id BIGINT NOT NULL,
  project_id BIGINT,
  customer_id BIGINT,
  note VARCHAR(500),
  status VARCHAR(20) DEFAULT 'PENDING' COMMENT 'PENDING|RESPONDED|TIMEOUT|IGNORED',
  response_content TEXT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  responded_at TIMESTAMP NULL,
```

### 5.4 新建 wdpp_boss_reports 表（存档）
```sql
CREATE TABLE wdpp_boss_reports (
  id BIGINT AUTO_INCREMENT PRIMARY KEY,
```
