## 1. 页面概述

| 属性 | 值 |
|------|-----|
| **路由** | `/wande-project/project` |
| **菜单层级** | 商务部 > 全球项目矿场 |
| **权限标识** | `wande:project:mine:list` / `query` / `add` / `edit` / `remove` / `export` / `insight`（v2.0新增洞察组权限） |
| **所属模块** | `module:fullstack` |
| **主要用户** | 吴耀（分配决策/全局洞察）、商务主管（团队管理/复盘）、业务员（执行跟进） |
| **业务定位** | 全球招投标项目线索发现+分配系统。覆盖国内招标、国际招标、幼儿园项目。不含经销商客户寻找（独立模块）和竞品情报（独立模块） |
| **v2.0 整体架构** | 一级分组（矿场执行 / 矿场洞察） + 二级Tab（执行6个 + 洞察4个） |

### 2.1.1 状态枚举（v2.0 新增 workStatus 字段）

替代旧的 11 种内部状态机，面向业务员操作的 6 种工作状态：

| 状态 key | 标签 | 颜色 | 含义 |
|---------|------|------|------|
| `pending_follow` | 待跟进 | 灰 | 分配后尚未联系 |
| `following` | 跟进中 | 蓝 | 已联系，正在推进 |
| `pending_quote` | 待报价 | 橙 | 客户已请求报价 |
| `quoting` | 报价中 | 紫 | 报价方案制作中 |
| `pending_decision` | 待客户决策 | 青 | 已提交方案，等客户答复 |
| `decelerated` | 已减速 | 红 | 流程放缓（需跟进但优先级降低）|

**流转规则**: 任意状态之间可切换（业务员主导），切换时记录 `mine_status_log`（旧状态/新状态/切换人/时间/备注）。

**与内部状态机的关系**: 内部技术状态（assigned/contacted/tracking/bid_preparing/...）仍保留用于系统流转；`workStatus` 是叠加的业务员视图。

**代码现状**: 🔴 需新建（数据库字段+Controller+前端下拉+拖拽）

### 2.7 Tab：转化漏斗（07-funnel.html）★ v2.0 新增

**原型路径**: `.github/docs/design/全球项目矿场/07-funnel.html`

- **组件**: 自研 `<FunnelChart>`（CSS 梯形） + 横向条形图 + 下钻表格
- **数据源**: 
  - `GET /wande/project/mine/funnel?dimension=&timeRange=&filter=`
  - `GET /wande/project/mine/funnel/stage-drill?stageKey=&page=&size=`
- **5阶段定义**: 早期 → 投资 → 谈判 → 确认 → 签约（横轴，所有项目共享；项目可跳过某些阶段）
- **展示指标**: 每阶段的 项目数 / 金额合计 / 转化率（vs上阶段）/ 停留天数
- **交互**: 
  - 切换时间范围（本月/上季/本年/自定义）
  - 维度切片（全公司/来源/区域/业务员/品类）
  - 点击阶段下钻到该阶段项目列表
  - 异常转化率（<35%）标 ⚠️
  - 导出 Excel 报表
- **告警规则**: 停留天数超过行业基准 150%（如谈判阶段 >60 天），标红提示
- **代码现状**: 🔴 需新建（#2255 #1532）

### 2.8 Tab：ROI 看板（08-roi.html）★ v2.0 新增

**原型路径**: `.github/docs/design/全球项目矿场/08-roi.html`

- **组件**: KPI卡片 + 来源渠道 ROI 排行表 + 6个月趋势图（CSS+SVG） + 异常提醒卡
- **数据源**:
  - `GET /wande/project/mine/roi/summary?timeRange=&currency=`

- **健康度分级**: 🟢优秀（ROI>5000%）/ 🟡良好（500%-5000%）/ 🔴低效（<500%）
- **交互**:
  - 时间/币种/来源类别筛选
  - 点击渠道行下钻该渠道项目
  - 导出 ROI 报表
- **告警规则**: 连续3个月零中标的渠道、获客成本环比上升>30% 的渠道 → 异常卡片提醒
- **代码现状**: 🔴 需新建（#2252），依赖成本数据录入机制
