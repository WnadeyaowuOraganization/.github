## 2. 页面布局

8大区块，4行2列网格布局：

```
┌────────────────────────────────────────────────────┐
│ 早上好，耀总 | 2026-04-09 | 数据更新: 3分钟前      │
├──────────────────┬─────────────────────────────────┤
│ 5.待办中心(340px) │ 1.矿场运营(KPI+漏斗+ROI)        │
├──────────────────┼─────────────────────────────────┤
│ 2.团队效能        │ 3.项目全景                       │
├──────────────────┼─────────────────────────────────┤
│ 4.财务概览        │ 6.复盘洞察                       │
├──────────────────┼─────────────────────────────────┤
│ 7.市场地图        │ 8.趋势雷达(政策信号)             │
└──────────────────┴─────────────────────────────────┘
```

### 2.1 待办中心（左上，最醒目）
- **定位**: 需要吴耀处理的事项集中展示
- **数据源**: 聚合多个模块的待办数据
- **内容**:
  - 待分配项目数（来自矿场，点击跳转矿场列表页筛选待分配）
  - 待确认验证数（来自矿场验证，有效/无效确认）
  - 待审批数（来自审批流引擎，报价/合同/报销）
  - 商务超时预警数（来自48h反馈机制）
  - AI推荐今日行动（来自NBO引擎 #1496）
  - AI建议卡片（最紧急的一条建议）

### 2.2 矿场运营（右上）
- **定位**: 项目矿场的运营概览
- **合并原3个独立页面**: mine-dashboard + #2255漏斗 + #2252 ROI + #2225验证 + #2318前期价值
- **内容**:
  - 6个KPI: 总项目/A级/B级/待分配/验证通过率/采集状态
  - 横向转化漏斗: 发现→分配→联系→跟进→中标（含各阶段转化率）
  - 信源ROI摘要: 最高ROI信源标注
  - 点击"查看全部项目"跳转全球项目矿场列表页

### 2.3 团队效能（左中）
- **定位**: 商务团队管理视角
- **合并**: #2320 商务跟进看板 + 销售记录体系数据
- **内容**: 商务人员表格（姓名/分配数/跟进数/本周拜访/平均响应/赢率/状态）
- **预警**: 超时商务红色标记 + 底部预警文字

### 2.4 项目全景（右中）
- **定位**: 在执行项目的进展管理
- **数据源**: 执行管理 + 项目中心 + EVM
- **内容**:
  - 4个KPI: 活跃项目数/总合同额/平均健康度/延期预警数
  - 项目阶段分布条（设计/生产/安装/验收/质保）
  - 延期预警项目列表（紧急/关注标签）

### 2.5 财务概览（左下）
- **定位**: 钱的流向
- **数据源**: 回款模块 + 合同管理 + 金蝶联动
- **内容**:
  - 4个KPI: 待回款/本月已回/逾期金额/平均利润率
  - 按项目回款进度条
  - 合同看板摘要（进行中/总额/即将到期）

### 2.6 复盘洞察（右下）
- **定位**: 赢/输分析
- **合并**: #2251 复盘洞察看板
- **内容**:
  - 3个KPI: 中标数/流失数/总赢率(vs上季度)
  - 流失原因分布条形图（价格/关系/产品/跟进慢）
  - AI洞察文字

### 2.7 市场地图（左底）
- **定位**: 全国/全球市场机会分布
- **合并**: #2250 区域热力图 + #2235 区域BI
- **内容**:
  - TOP10省份热度色块（颜色越深=项目越多）
  - 蓝海/红海区域标记
  - 国际市场摘要

### 2.8 趋势雷达（右底）
- **定位**: 政策信号预警——未来会发生什么
- **合并**: #2239 政策信号Feed
- **内容**:
  - 政策信号列表（高/中/低关联度标签）
  - 每条信号: 标题 + 省份/国家标签 + 预计影响 + 矿场已关联项目数
  - 点击"全部信号"跳转完整政策信号页面

## 3. API契约（新增）

### GET /wande/cockpit/boss/overview
- **权限**: `wande:cockpit:boss:view`
- **返回**: 8大区块的聚合数据
  - `mineOps`: {totalCount, aGrade, bGrade, pendingAssign, verifyRate, crawlerStatus, funnelData[], topSourceROI}
  - `teamPerf`: {members[]{name, assigned, tracking, weekVisits, avgResponse, winRate, status}}
  - `projectPanorama`: {activeCount, totalAmount, avgHealth, delayWarnings[]}
  - `finance`: {pendingPayment, monthCollected, overdueAmount, avgProfitRate, projectProgress[], contractSummary}
  - `todoCenter`: {pendingAssign, pendingVerify, pendingApproval, overtimeWarnings, nboRecommendations[]}
  - `review`: {wonCount, lostCount, winRate, winRateChange, lossReasons[]}
  - `marketMap`: {provinceRanking[], blueOcean[], redOcean[], international[]}
  - `trendRadar`: {signals[]{title, region, relevance, impact, linkedProjectCount, discoveredAt}}

### GET /wande/cockpit/boss/todo-count
- **说明**: 轻量接口，仅返回待办总数（用于菜单徽标）
- **返回**: `{total, breakdown: {assign, verify, approval, overtime, nbo}}`

## 4. 数据库设计

不新建表。耀总驾驶舱是纯聚合视图，数据来自：
- 矿场: `wdpp_project_mine` + `wdpp_project_mine_source` + `wdpp_project_mine_assign_log`
- 销售: `activity_logs` + 用户表
- 项目: 执行管理表 + 项目中心表
- 财务: 回款表 + 合同表 + 金蝶API
- 审批: 审批流引擎表
- 政策: 政策信号采集表（pipeline侧）
