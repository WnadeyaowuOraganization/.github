# ADR-002: 共享Volume缓存策略

## 状态
- **日期**: 2026-04-11
- **状态**: 已接受
- **决策者**: 排程经理、研发经理

## 背景

20个编程CC目录各自独立编译，导致：
- 重复下载Maven依赖（每个kimi约2GB）
- 重复安装pnpm依赖（每个kimi约1GB）
- 磁盘I/O浪费，编译时间长（5-8分钟）

## 决策

采用"**只读共享 + 本地写入**"策略，共享依赖缓存。

## 方案对比

| 方案 | 优点 | 缺点 | 选择 |
|------|------|------|------|
| 各自独立 | 完全隔离 | 磁盘浪费40GB | ❌ |
| 完全共享 | 最省空间 | 并发写入冲突 | ❌ |
| 远程缓存 | 可跨机器 | 需额外基础设施 | ❌ |
| **只读共享+本地写入** | **平衡空间与并发** | 实现简单 | ✅ |

## 架构设计

```
/home/ubuntu/.m2/repository        ← 共享Maven依赖（只读）
/home/ubuntu/.pnpm-store           ← 共享pnpm缓存（只读）
/tmp/cc-maven-cache-kimi{N}/       ← 本地写入缓存
/apps/wande-ai-backend-kimi{N}/    ← 各kimi独立产物
```

## 实施步骤

1. 创建共享目录
2. 设置权限（ubuntu用户）
3. 修改run-cc.sh注入环境变量
4. 验证缓存命中率

## 后果

### 正向影响
- 磁盘占用减少60%（40GB→16GB）
- 编译时间减少60%（5-8分钟→2-3分钟）
- 网络下载减少80%

### 风险及缓解
| 风险 | 缓解措施 |
|------|----------|
| 并发写入冲突 | 只读共享+本地写入策略 |
| 缓存失效 | 版本号变化自动失效 |
| 磁盘空间增长 | 定期清理未使用依赖 |
