## § 1. 页面概述

### 1.1 体系定位

询盘报价体系是万德AI平台商务部核心工作流，覆盖从客户询盘到订单结清的完整商业闭环。三大页面共享同一个 `business_type` 枚举（`direct` 直销 / `dealer` 经销 / `international` 国贸），在统一 UI 框架下差异化渲染各业务线专属字段与功能模块，替代分散在 Excel / 明道云 / 邮件附件中的线下流程。

**三线统一（business_type）设计原则：**

| business_type | 中文名 | 特殊功能 | 颜色标识 |
|---|---|---|---|
| `direct` | 直销 | 标准报价 + 投标附件 | 灰色条 `#e5e7eb` |
| `dealer` | 经销 | 折扣率管控 + 经销合同字段 + 信用额度 | 黄色条 `#fef9c3` |
| `international` | 国贸 | 汇率锁汇 + PI 生成 + 出口单证（CI/PL/BL/CO/FormE）| 紫色条 `#ede9fe` |

**三大角色：**

| 角色 | 英文标识 | 核心权限 |
|---|---|---|
| 业务员 | `salesperson` | 创建询盘、编辑报价（基础字段）、查看自己名下数据 |
| 商务主管/老板 | `manager` / `boss` | 审批、查看全量数据、成本模型、信用额度管理 |
| 国贸专员 | `intl_specialist` | 锁汇操作、PI 生成、出口单证管理、发货跟踪 |

### §3.1 询盘 API

#### GET /api/trade/inquiries

列表查询，支持筛选、分页、角色过滤。

**Query 参数：**

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `page` | int | 否 | 页码，默认 1 |
| `size` | int | 否 | 每页条数，默认 20，最大 100 |
| `view` | string | 否 | `kanban`（按 stage 分组返回）/ `table`（分页返回）|

| `biz` | string | 否 | `direct,dealer,international`（逗号分隔多选）|

| `source` | string | 否 | 来源渠道：`website,exhibition,referral,bid_platform` |
| `date_from` | string | 否 | `YYYY-MM-DD` |
| `date_to` | string | 否 | `YYYY-MM-DD` |

## § 6. H5 适配方案

> Sprint-3 延后，H5 版本暂不在本次迭代交付，仅说明设计思路。

**规划路由：**
- `/h5/crm/inquiry-workbench` — 询盘工作台移动端（仅看板视图）
- `/h5/crm/quote/:id` — 报价审批查看（只读 + 审批操作）

**组件库：** Vant 4（配合项目已有 H5 基础框架）

**核心适配思路：**

1. **询盘看板：** 业务员移动端主要用于外出拜访时快速查询询盘状态，保留看板视图（横向滚动卡片），不实现表格视图和时间线视图。
2. **报价审批：** 商务主管/老板在手机端收到企微审批通知后，跳转 H5 页面查看报价摘要（成本模型 + 金额 + 折扣），点击「通过」/「驳回」+ 填写意见。
3. **不实现：** 报价编辑器、锁汇操作、单证管理（复杂操作保留 PC 端）。
4. **布局方案：** 独立 H5 页面（非响应式 PC，避免复杂 layout 的移动端降级问题）。
5. **身份认证：** 复用微信网页授权，与企微 JSSDK 打通。

## § 7. PageGuide 内容

> PageGuide 在用户首次访问页面时弹出，提供快速上手指引。

### §7.1 询盘工作台 PageGuide

**这是什么：**
询盘工作台是你管理所有客户询盘的核心界面，支持直销、经销、国贸三类业务在同一看板上统一呈现。

**解决什么问题：**
告别分散在 Excel 表格和邮件附件中的询盘记录，所有询盘状态、金额、跟进人一目了然，再也不会漏掉任何一个商机。

**快速上手 3 步：**
1. 在右上角选择「角色」和「业务线」筛选你关心的询盘范围；
2. 拖拽卡片到不同列即可推进询盘阶段，系统自动记录变更历史；
3. 点击卡片上的「📝报价」快捷按钮直接跳转到报价生成页。

---

### §7.2 报价生成页 PageGuide

**这是什么：**
报价生成页是从产品选品到报价提交的一站式工作台，支持多版本报价对比、成本利润测算（主管可见）、国贸汇率锁汇，以及审批完成后一键生成 PI 形式发票。

**解决什么问题：**
替代需要手工维护的 Excel 报价单，避免版本混乱、成本核算不准、汇率损益难追踪等问题，并自动对接审批流让主管实时看到每一版报价的变化。

**快速上手 3 步：**
1. 从左侧选品目录双击产品加入明细，在右侧直接修改数量和折扣；
2. 填写完成后点击「提交审批」，系统根据金额和折扣自动确定审批级别；
3. 审批通过后（国贸客户），点击「立即生成 PI」即可获得双语 PDF 形式发票。

---

### §7.3 订单跟踪看板 PageGuide

**这是什么：**
订单跟踪看板将每一笔成交订单从 PI 确认到最终结清的全流程可视化，国贸订单额外展示发货信息、单证进度（CI/PL/BL/CO/FormE）和汇率损益。

**解决什么问题：**
让业务员随时掌握每笔订单处于哪个阶段、是否超期、钱是否收回来、国贸货到哪里了，替代邮件+电话追踪的低效方式。

**快速上手 3 步：**
1. 通过顶部「角色」和「业务线」筛选聚焦你关注的订单；
2. 点击国贸订单卡片打开右侧详情抽屉，可查看发货状态和逐一下载/上传出口单证；
3. 订单完成后点击「推进到下一阶段」直至「结清」，国贸订单结清时系统自动计算汇率损益。
