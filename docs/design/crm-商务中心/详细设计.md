## 1. 业务概述

### 1.1 模块定位

万德CRM商务中心是商务团队的核心工作台，覆盖**直销甲方、国内经销商、国际客户**三条业务线的全生命周期管理：

```
询盘 → 可行性评估 → 商机创建 → 7阶段跟进 → 合同签署 → 回款跟踪 → 结案
```

### 1.2 三线差异化

| 维度 | 直销甲方 | 国内经销商 | 国际客户 |
|------|---------|-----------|---------|
| 核心维度 | 以项目为核心 | 以客户为核心 | 以客户为核心 |
| 商机来源 | 招标公告/矿场 | 经销商上报 | 展会/外展 |
| 跟进重点 | 招标时间节点 | 经销商培育/返点 | 汇率/文化差异 |
| 回款货币 | 人民币 | 人民币 | 美元/欧元/多币种 |
| 记录视角 | 项目中心视角 | 客户中心视角 | 客户中心视角 |

### 1.3 商机7阶段（照搬明道云）

```
新建(N) → 商务接触(C) → 招标准备(B) → 签约中(S) → 回款中(P) → 验收(A) → 结案(D)
```

### 1.4 边界说明（不在CRM内）

- 过会管理 → 耀总驾驶舱
- 竞品情报 → 独立板块
- 外出拜访打卡 → Sprint-2
- 直销/经销月度+年度业绩 → 耀总驾驶舱
- 经销商发现（新经销商招募） → 独立于CRM

## 3. 页面01 — 仪表盘 (01-dashboard.html)

### 3.1 页面概述

- **路由**: `/wande/crm/dashboard`
- **菜单层级**: CRM商务中心 > 仪表盘
- **权限标识**: `crm:dashboard:view`
- **主要用户**: 商务人员（个人视角）、销售主管（团队视角）

### 3.2 页面布局

#### 顶部KPI统计条（内联式，非卡片）

| 指标 | 字段 | 颜色 |
|------|------|------|
| 本月新增商机 | `month_new_opportunity_count` | #333 |
| 跟进中 | `active_opportunity_count` | #1890ff |
| 本月已签合同 | `month_signed_contract_count` | #52c41a |
| 逾期未跟进 | `overdue_followup_count` | #ff4d4f |
| 本月回款 | `month_payment_amount` | #52c41a |

**设计约束**: KPI以内联stat bar形式展示，**禁止使用卡片**。

#### 今日待办区

字段：`todo_list[]`，每条带：类型标签（回款/跟进/报价/审批）、标题、关联商机/客户、截止时间、状态。

操作：行内 [处理] 按钮，点击跳转对应页面。

#### 评估面板（可行性评估）

- **触发时机**: 商机分配给商务时，需做 Go/No-Go 决策
- **字段**: 矿场来源信息（项目名/省市/预算/开放时间）、评估结论（Go/No-Go）、评估理由
- **矿场信息同步**: 从项目矿场模块同步，字段包含 project_mine_id/项目名称/省市/预算区间/开放时间

### 3.3 API

```
GET /api/crm/dashboard/overview
  → { kpi: {...}, todos: [...] }

GET /api/crm/dashboard/assessment/pending
  → [{ opportunity_id, project_mine_id, project_name, location, budget_range, open_date }]

POST /api/crm/opportunity/{id}/assessment
  body: { decision: "go"|"no_go", reason: string }
```

表格列：

| 列名 | 字段 | 说明 |
|------|------|------|
| 商机名称 | `opportunity_name` | 点击进入详情 |
| 客户 | `customer_name` | |
| 阶段 | `stage` | 7阶段标签 |
| 金额 | `amount` | |
| 预计签约日 | `expected_close_date` | 超期变红 |
| 负责人 | `owner_name` | |

| 最近跟进 | `last_followup_date` | 超7天变橙 |

#### 看板视图

7列 Kanban，每列显示该阶段商机卡片（商机名/客户/金额/最近跟进日/负责人），支持拖拽推进阶段。

### 5.3 商机7阶段定义

| code | 名称 | 说明 |
|------|------|------|
| N | 新建 | 刚录入，未正式跟进 |
| C | 商务接触 | 已联系，需求确认中 |
| B | 招标准备 | 进入招投标流程 |
| S | 签约中 | 合同谈判/审批 |
| P | 回款中 | 合同已签，等待回款 |
| A | 验收 | 设备到场验收 |
| D | 结案 | 全部完成 |

### 5.4 15种商机来源分类（从明道云照搬）

矿场发现、招标公告、客户主动来访、展会获客、经销商上报、老客户复购、内部转介绍、政府关系、网络询盘、电话开发、合作伙伴、行业协会、媒体报道、社交媒体、其他

### 5.5 API

```
GET /api/crm/opportunity/list?stage=&owner=&biz_line=&page=&size=
GET /api/crm/opportunity/kanban  → 按阶段分组的数据
POST /api/crm/opportunity
PUT /api/crm/opportunity/{id}/stage  body: { stage: "C"|"B"|"S"|"P"|"A"|"D" }
```

### 6.1 页面概述

- **路由**: `/wande/crm/opportunity/:id`
- **权限标识**: `crm:opportunity:detail`
- **布局模式**: 左摘要（240px）+ 右侧多Tab（10个Tab）

### 6.2 左侧摘要区

固定字段：商机名称、阶段（带进度条）、客户名、合同金额、预计签约日、负责人、来源、创建时间

| # | Tab名称 | 内容 |
|---|---------|------|
| 1 | 基本信息 | 完整商机字段（从明道云照搬） |
| 2 | 跟进记录 | 时间线形式，商务填写的跟进日志 |
| 3 | 设计单 | 关联设计单（from D3模块） |
| 4 | 报价 | 报价单列表，可新建/查看 |
| 5 | 合同 | 合同文件（含盖章状态）Sprint-1只做基础创建 |
| 6 | 回款 | 回款节点+收款记录 |
| 7 | 投标HZ | 投标申请+标书文件 |
| 8 | 资料 | 附件上传/下载 |
| 9 | 流程监控 | 审批流状态（提交/审批/通过/驳回） |
| 10 | 变更日志 | 字段变更历史（自动记录） |

### 6.4 可行性评估（放弃跟进 Disqualification）

当商务在跟进中决定放弃时，点击[放弃跟进]弹出面板：

字段：
- 放弃原因（必填，下拉：预算不足/竞争对手中标/客户需求变化/关系资源不足/其他）
- 详细说明（选填）
- 确认复选框："我确认已知晓放弃后商机将关闭"

操作后：商机阶段变为 `closed_lost`，在管道列表中隐藏（可通过筛选查看）。

### 6.5 矿场信息同步

若商机来源 = 矿场发现，基本信息Tab显示"矿场来源"区块：
字段：project_mine_id、项目名、省市、预算区间、开放时间、矿场评分、中标概率预测

### 6.6 API

```
GET /api/crm/opportunity/{id}  → 完整商机详情
GET /api/crm/opportunity/{id}/followups  → 跟进记录列表
POST /api/crm/opportunity/{id}/followup  body: { content, follow_date, next_action }
```

### 8.1 页面概述

- **路由**: `/wande/crm/records`
- **权限标识**: `crm:record:list`
- **核心设计**: `activity_logs` 聚合引擎，同时关联 project_id + customer_id + user_id，支持四视角查询

### 8.2 四视角Tab

| Tab | 视角 | 说明 |
|-----|------|------|
| 全部 | 全局 | 本人或团队所有记录 |
| 按项目 | project_id | 某个项目下的所有操作 |
| 按客户 | customer_id | 某个客户下的所有互动 |
| 按成员 | user_id | 某个商务的所有记录（管理员可看全员） |

### 8.3 主表格

| 列名 | 字段 | 说明 |
|------|------|------|
| 时间 | `log_time` | |
| 类型 | `log_type` | 跟进/报价/合同/回款/投标/设计单 |
| 内容摘要 | `content` | |
| 关联商机 | `opportunity_name` | |
| 关联客户 | `customer_name` | |
| 操作人 | `operator_name` | |
| 操作 | — | [详情] |

### 8.4 周报/月报入口

页面右上角：[填写周报] [查看月报]

**周报**（每周四截止）：
- 填写本周跟进总结
- 异常项标记（逾期/卡住的商机）
- 提交后推送给销售主管

**月报**（每月3号截止）：
- 自动聚合本月 activity_logs 生成草稿
- 商务审核后提交
- 维度：新增商机数/签约额/回款额/丢单分析

### 8.5 API

```
GET /api/crm/activity-log/list?view=project|customer|user&ref_id=&type=&page=&size=
POST /api/crm/activity-log
GET /api/crm/weekly-report/current  → 本周周报状态
POST /api/crm/weekly-report  body: { week_start, content, abnormal_items }
GET /api/crm/monthly-report?year=&month=  → 月报详情（含自动聚合草稿）
```

### 8.6 数据库

表名：`crm_activity_log`
