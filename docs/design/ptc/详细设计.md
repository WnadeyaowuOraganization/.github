### 三类产品PLM操作模式（核心章节）

这是本设计文档的核心策略章节。万德产品分三类，每类在PLM中的操作路径不同：

#### 1.1 标准化产品 — MTS (Make-to-Stock)

**定义**: 固有模具件，不可改几何。PE板功能件(49款)、娃娃家(37款)、转椅(9款)、滚塑摇马等。

**BOM策略**: 固定BOM（Variant BOM）
- 一个零件号 = 一个固定BOM，不允许修改
- 版本状态机：Draft → Review → Released → Obsolete
- Released状态下任何修改必须通过ECO创建新版本

**PLM操作路径**:
```
销售选型(产品目录) → 系统直接输出固定BOM → 成本自动Roll-up → 报价 → 下单
```

**涉及Tab**: ② 零件主数据（Part Master管理） + ⑧ 安全合规（证书绑定BOM版本）

**CC开发要点**:
- Part Master列表页需支持按"产品类型=MTS"筛选
- BOM创建时如果product_type=MTS，锁定BOM编辑（只能通过ECO修改）
- 合规证书与BOM版本强绑定，版本变更自动触发"需重新认证"警告

---

#### 1.2 半标产品 — CTO (Configure-to-Order)

**定义**: 有标准平台+可调参数。秋千(改高度/座位数)、滑梯组合(改平台高度/滑道类型/护栏材质)、攀爬架(改尺寸/网格间距)等。万德最大出货量的产品类型。

**BOM策略**: 超级BOM（Configurable BOM）+ 配置规则
- 一个产品系列 = 一个超级BOM，定义所有可选参数和规则
- 客户选配后，系统自动从超级BOM派生出具体的EBOM实例
- 三种规则类型：N选1（滑道类型选一种）/ 数量型（平台数量1-4）/ 参数化（平台高度1.2-2.0m步进0.3m）
- 约束规则：如"螺旋滑道需要平台高度≥1.5m"

**PLM操作路径**:
```
销售进入配置器(Tab⑤) → 选产品系列 → 选配参数
  → 规则引擎实时过滤无效组合 → 自动派生EBOM实例
  → 成本自动Roll-up + 利润率计算（红线校验28%）
  → 技术确认(Tab⑥，可选) → 报价 → 下单
```

**涉及Tab**: ③ BOM管理中心（超级BOM Tab）+ ⑤ 产品配置器 + ⑥ 技术确认中心

**CC开发要点**:
- 超级BOM规则引擎（#3388）是核心，支持三种规则类型 + 约束校验
- 配置器前端（#3397）需实时调用规则引擎API，动态渲染可选项
- 派生EBOM（#3389）不创建新零件号，而是从超级BOM中选择已有零件组合
- 利润红线校验（#3392）：低于28%自动标红警告，低于20%禁止提交需管理员审批
- **非标参数转正机制**：同一非标配置被使用≥3次，系统自动识别并建议加入超级BOM规则（详见「非标转CTO Spec」）

---

#### 1.3 纯非标产品 — ETO (Engineer-to-Order)

**定义**: 主题乐园整体方案、海盗船异形定制、客户特殊场地定制造型。每个订单需要设计师从头设计或大幅修改。

**BOM策略**: 项目BOM（Project BOM）
- 每个订单创建独立的项目BOM
- 尽量从标准零件库拉取已有零件（目标：80%复用标准模块，20%新设计件）
- 新设计件必须经过ECO流程审批后才能入库

**PLM操作路径**:
```
客户需求 → 设计师D3工作台(Tab⑦)出图
  → BOM推送PLM(Tab③)创建项目BOM
  → 新设计件走ECO审批(Tab④，A/B/C分级)
  → 技术确认(Tab⑥)胡总确认参数
  → 合规校验(Tab⑧)
  → 项目BOM锁定 → 报价（实时成本计算，非审批拦截）→ 下单
```

**涉及Tab**: ⑦ D3设计工作台 → ③ BOM管理中心（项目BOM Tab）→ ④ ECO工程变更 → ⑥ 技术确认 → ⑧ 安全合规

**CC开发要点**:
- D3推送BOM时（#3390），自动标记每个零件来源：已有Part Master / 新设计件
- 新设计件自动创建ECO申请，走审批后才能入Part Master
- 项目BOM中的复用率统计：自动计算"标准件占比"并显示（目标≥80%）
- **非标→CTO持续转化**：追踪ETO项目中的高频配置模式，≥3次自动建议转入超级BOM

---

#### 1.4 三类产品操作路径汇总表

| 维度 | MTS标准化 | CTO半标 | ETO纯非标 |
|------|----------|---------|----------|

### Tab④ ECO工程变更 — 04-eco-center.html

对应Issue: #3385 #3386 #3387 #3396

#### [KPI统计行]

#### [筛选栏]
- **组件**: `a-input-search` + `a-select`(级别A/B/C) + `a-select`(状态) + `a-range-picker` + `a-button`[发起ECO]

#### [ECO列表]
- **组件**: `a-table`
- **列**: ECO编号 | 变更标题 | 级别(A/B/C tag) | 触发源(MTS版本升级/CTO规则变更/ETO新件入库) | 影响零件数 | 影响BOM数 | 预估成本影响 | 发起人 | 状态 | 操作[详情][审批]
- **级别标签色**: A=红 B=橙 C=灰

#### [ECO详情展开]
- **变更原因+描述**: 富文本
- **影响分析表格**: 受影响零件 + 受影响BOM + 变更前后对比
- **审批流可视化**: 横向步骤条
  - A级(重大): 发起人 → 技术评审(胡总) → 成本评审(财务) → 总经理会签 → 执行
  - B级(中等): 发起人 → 技术评审 → 执行
  - C级(轻微): 发起人 → 自动执行
- **执行状态**: 零件版本更新进度 + BOM版本递增状态

#### ECO与三类产品的关系
| 产品类型 | ECO触发场景 | ECO级别 | 频率 |
|---------|-----------|---------|------|
| MTS | 材质升级、安全标准更新 | 通常A级 | 低（每年几次） |
| CTO | 超级BOM规则变更、新增配置选项 | 通常B级 | 中（每月几次） |
| ETO | 新设计件入库、项目BOM变更 | 通常B/C级 | 高（每个项目） |

#### 三级边界规则
```
✅ Always:
- ECO编号自动生成：ECO-{年份}-{4位序号}
- A级ECO必须三人会签（技术+财务+总经理），不可跳过
- C级ECO自动执行，不需要人工审批
- ECO审批通过后自动触发零件版本更新+BOM版本递增
- ECO必须标注触发源（MTS版本升级/CTO规则变更/ETO新件入库）

⚠️ Ask First:
- ECO审批人配置（目前硬编码：A级=胡总+财务+总经理，是否需要动态配置）
- ECO是否需要和企微审批打通（复用审批体系？）

🚫 Never:
- 不允许绕过ECO直接修改Released状态的零件或BOM
- 不允许关闭有未执行变更的ECO
```

**代码现状**: 
- 🟡 改造: 已有审批引擎（审批体系Sprint-2），ECO审批可复用
- 🔴 新建: plm_eco表 + 影响评估引擎 + ECO执行引擎（自动更新版本）

### Tab⑤ 产品配置器 — 05-configurator.html

对应Issue: #3391 #3392 #3393 #3397

#### [左栏 — 选配面板]（30%宽度）

  - 参数化规则 → `a-select`(预设值) 或 `a-slider`(连续范围)
  - 多选规则 → `a-checkbox-group`
- **约束冲突提示**: 实时红色文字提示（如"螺旋滑道需要平台高度≥1.5m"）
- **操作**: [生成配置方案] 蓝色按钮

#### [中栏 — 3D预览区]（40%宽度）
- **组件**: Three.js GLB模型加载器（CC实现时使用Three.js）
- **降级方案**: 3D未就绪时显示产品系列渲染图
- **工具栏**: 旋转 | 缩放 | 爆炸图 | 重置视角
- **选配结果摘要**: 文字描述当前配置

#### [右栏 — BOM+报价]（30%宽度）
- **生成的EBOM表格**: `a-table`
  - 列: 零件编码 | 名称 | 数量 | 单价 | 小计
  - 数据源: `POST /api/ptc/configurator/resolve` (提交选配参数，返回EBOM实例)
- **报价汇总**:
  - 材料成本 / 加工成本 / 安装费 / 利润率(百分比+颜色) / 建议报价
  - 利润率≥35%=绿色 / 28-35%=橙色 / <28%=红色(禁止提交，需管理员审批)
- **操作按钮**: [导出报价单] [发送技术确认] [保存配置]

#### 非标参数转正机制（关键功能）
- 详见「非标转CTO交互Spec」文档
- 在配置器中，如果销售输入了不在超级BOM规则内的参数值（"非标参数"），系统记录该参数
- 当同一非标参数被使用≥3次，系统在Tab⑤右栏底部弹出提示："该配置已被使用3次，建议加入超级BOM标准规则"
- 管理员点击[转正]后，自动创建ECO申请，走B级审批，通过后新参数加入超级BOM规则

#### 三级边界规则
```
✅ Always:
- 配置器必须实时调用规则引擎API校验，不允许前端硬编码规则
- 利润红线28%是硬约束，低于28%必须红色警告+禁止普通销售提交
```

## 3. API契约

### 3.1 零件主数据 API

#### GET /api/ptc/parts
- **参数**: keyword(string) / category(enum) / status(enum: Draft|Review|Released|Obsolete) / material(string) / productType(enum: MTS|CTO|ETO) / page(int) / size(int)
- **返回**: { total, list: [{ id, partCode, name, category, material, version, status, productType, bomRefCount, creator, updatedAt }] }

#### POST /api/ptc/parts
- **参数**: { name, category, material, productType, specs: {} }
- **返回**: { id, partCode(自动生成) }
- **规则**: partCode由编码引擎自动生成，格式WD-XX-XX-XXXX

#### PUT /api/ptc/parts/:id/status
- **参数**: { targetStatus: "Review"|"Released"|"Obsolete", reason }
- **规则**: Released→修改时自动创建新版本+ECO

### 3.2 BOM API

#### GET /api/ptc/bom
- **参数**: type(enum: EBOM|SuperBOM|MBOM|ProjectBOM) / keyword / status / page / size
- **返回**: { total, list: [{ id, bomCode, productName, version, partCount, totalCost, source(MTS|CTO|ETO), status }] }

#### GET /api/ptc/bom/:id/tree
- **返回**: 递归BOM树 { id, partCode, name, qty, unitPrice, subtotal, whereUsedCount, source, children: [...] }

#### GET /api/ptc/bom/:id/diff
- **参数**: v1(int) / v2(int)
- **返回**: { added: [...], removed: [...], modified: [{ part, before, after }] }

#### POST /api/ptc/bom/:id/rollback
- **参数**: { targetVersion }
- **规则**: 创建新版本（内容=目标版本），不是真正回退

### 3.3 超级BOM规则 API

#### GET /api/ptc/super-bom/:id/rules
- **返回**: [{ id, ruleName, ruleType(N选1|数量型|参数化), parameter, options/range, constraints, status }]

#### POST /api/ptc/super-bom/:id/rules
- **参数**: { ruleName, ruleType, parameter, options, constraints }
- **规则**: 新增/修改规则必须通过ECO（B级）

#### POST /api/ptc/configurator/resolve
- **参数**: { seriesId, selections: { ruleId: selectedValue, ... } }
- **返回**: { ebomItems: [...], cost: { material, process, install, total }, profitRate, suggestedPrice, warnings: [...] }

### 3.4 ECO API

#### POST /api/ptc/eco
- **参数**: { title, description, level(A|B|C), triggerSource(MTS_UPGRADE|CTO_RULE_CHANGE|ETO_NEW_PART), affectedParts: [...], affectedBoms: [...], estimatedCostImpact }
- **返回**: { id, ecoCode(自动: ECO-YYYY-NNNN) }

#### POST /api/ptc/eco/:id/approve
- **参数**: { action(approve|reject), comment }

#### POST /api/ptc/eco/:id/execute
- **规则**: 审批通过后触发，自动更新零件版本+BOM版本递增+通知

### 3.5 技术确认 API

#### POST /api/ptc/confirm
- **参数**: { productId, projectName, bomVersion, params: [{ name, clientRequirement, actualSpec }] }
- **返回**: { id, confirmCode(TC-YYYY-NNNN) }

#### PUT /api/ptc/confirm/:id/approve
- **参数**: { action(approve_all|partial_approve|reject), comment, confirmedParams: [...] }
- **规则**: approve_all → 自动锁定BOM版本

### 3.6 非标转CTO API（新增）

#### GET /api/ptc/configurator/non-standard-stats
- **返回**: [{ paramKey, paramValue, usageCount, firstUsedAt, lastUsedAt, projects: [...] }]
- **筛选**: usageCount >= 3 的非标参数列表

#### POST /api/ptc/configurator/promote-to-cto
- **参数**: { paramKey, paramValue, targetSuperBomId, ruleType }
- **规则**: 自动创建B级ECO → 审批通过后加入超级BOM规则

### 3.7 安全合规 API

#### GET /api/ptc/compliance/certificates
- **参数**: standard(EN1176|GB19272|ASTM) / status / keyword / page / size

#### POST /api/ptc/compliance/certificates/:id/bind-bom
- **参数**: { bomId, bomVersion }

#### POST /api/ptc/compliance/ai-report
- **参数**: { productId, standards: [...], language }
- **返回**: { reportUrl(PDF), summary }
