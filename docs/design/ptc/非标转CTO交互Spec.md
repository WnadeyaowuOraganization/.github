# 非标参数转正机制(ETO→CTO) — 交互Spec

> 所属模块: 产品技术中心(PLM) > Tab⑤ 产品配置器
> 对应Issue: #3389（超级BOM→实例EBOM解析API — 非标参数转正机制）
> 创建日期: 2026-04-13
> 状态: 待开发
> 设计依据: [Tacton CTO覆盖率提升最佳实践](https://www.tacton.com/cpq-blog/mastering-mass-customization-increasing-configure-to-order/) — "追踪重复ETO请求，达到X次后评估CTO纳入"

---

## 1. 业务背景

万德产品分三类（MTS/CTO/ETO），其中CTO半标产品通过超级BOM配置规则自动选配。但实际业务中，销售经常在配置器中输入超级BOM规则范围外的参数值（"非标参数"），这些参数只能走ETO流程（设计师出图）。

**核心问题**: 很多非标参数其实是高频需求，只是还没被纳入超级BOM规则。如果同一个非标参数反复出现3次以上，说明它应该被"转正"为CTO标准配置。

**目标**: 建立自动化的非标参数识别→建议转正→ECO审批→规则入库闭环，持续扩大CTO覆盖率，减少ETO工作量。

---

## 2. 核心概念

| 概念 | 定义 | 举例 |
|------|------|------|
| 标准参数 | 已在超级BOM规则中定义的参数值 | 平台高度1.2m/1.5m/1.8m/2.0m |
| 非标参数 | 超出超级BOM规则范围的参数值 | 平台高度2.3m（规则中没有） |
| 非标配置 | 包含至少一个非标参数的配置方案 | 超级滑梯-螺旋-平台高度2.3m |
| 转正(Promote) | 将高频非标参数加入超级BOM规则 | 2.3m加入平台高度选项 |
| 转正阈值 | 触发转正建议的最低使用次数 | 默认3次（可配置） |

---

## 3. 识别规则

### 3.1 什么算"非标参数"

当销售在配置器(Tab⑤)中提交选配方案时，系统逐项检查每个参数值：

```
对于每个选配参数:
  if 参数值 ∈ 超级BOM规则的options/range:
    → 标准参数，正常处理
  else:
    → 非标参数，记录到 plm_config_log.non_standard_params
    → 同时更新 plm_non_standard_param 统计表的 usage_count + 1
```

**非标参数的判定细节**:

| 规则类型 | 标准范围 | 非标判定条件 |
|---------|---------|------------|
| N选1 | options: [直滑, 螺旋, 波浪] | 选了options之外的值（如"双螺旋"） |
| 数量型 | range: {min:1, max:4, step:1} | 值超出min-max范围（如5个平台） |
| 参数化 | range: {min:1.2, max:2.0, step:0.3} | 值超出范围或不在步进上（如2.3m或1.4m） |
| 多选 | options: [攀爬网, 消防滑杆, 传声筒] | 选了options之外的附加模块 |

### 3.2 记录粒度

非标参数按 `param_key + param_value` 组合统计，不区分产品系列：

- `param_key` = 规则参数名（如"平台高度"）
- `param_value` = 具体非标值（如"2.3m"）
- 同一个 param_key+param_value 在不同项目/不同销售使用，都累加到同一条统计记录

### 3.3 排除条件

以下情况不计入非标统计：
- 同一个销售对同一个项目重复提交（去重：同一user_id + 同一project_name + 同一param_key+value，24小时内只计1次）
- 已被标记为"已忽略"(DISMISSED)的非标参数
- 已被成功转正(PROMOTED)的参数

## 4. 触发条件

### 4.1 转正建议触发

当 `plm_non_standard_param.usage_count >= 转正阈值(默认3)` 且 `status = TRACKING` 时，系统触发转正建议。

**触发时机**: 每次配置器提交后实时检查（不是定时任务）

### 4.2 通知方式（三级）

| 级别 | 条件 | 通知方式 | 通知对象 |
|------|------|---------|---------|
| L1 提示 | usage_count = 3（首次达标） | 配置器页面内弹出提示条 + 总览仪表盘待办 | 当前操作的销售 + PLM管理员 |
| L2 推荐 | usage_count >= 5 | 企微通知 | PLM管理员(胡总) |
| L3 强推荐 | usage_count >= 10 | 企微通知 + 总览仪表盘红色待办 | PLM管理员(胡总) + 吴耀 |

### 4.3 转正阈值配置

- 默认值: 3次
- 配置位置: 系统设置 > PLM参数 > 非标转正阈值
- 允许范围: 2-10次
- 修改权限: 仅超管

## 5. UI交互流程

### 5.1 销售端 — 配置器中触发提示

**场景**: 销售在Tab⑤配置器中选配参数，某个非标参数累计达到3次。

**交互流程**:

```
Step 1: 销售提交配置方案
  → 系统检测到1个非标参数（平台高度2.3m）
  → 弹出黄色提示条（a-alert type=warning）:
    "⚠️ 参数「平台高度 = 2.3m」不在标准配置范围内，
     将作为非标方案处理（需设计师确认）。
     该参数已被使用 3 次。[建议转为标准配置 →]"

Step 2: 销售点击 [建议转为标准配置 →]
  → 弹出确认弹窗（a-modal）:
    标题: 建议将非标参数转为标准配置
    内容:
      参数名: 平台高度
      非标值: 2.3m
      累计使用: 3次（来自2个不同项目，2位销售）
      最近使用: 合肥体育公园二期（张三）、成都天府新区（李四）、杭州滨江公园（张三）
      
      如果转正成功，未来销售可直接在配置器中选择2.3m，无需走非标流程。
      
    操作: [提交转正申请] [暂时跳过]

Step 3: 点击 [提交转正申请]
  → 系统自动创建B级ECO:
    标题: "超级BOM规则扩展 — 平台高度新增2.3m选项"
    级别: B级
    触发源: CTO_RULE_CHANGE
    关联非标参数: plm_non_standard_param.id
  → 提示: "已创建ECO申请 ECO-2026-XXXX，等待技术评审(胡总)审批。"
  → plm_non_standard_param.status 改为 PROMOTING

Step 4: 当前非标方案继续走ETO流程（不阻塞）
```

### 5.2 管理员端 — 总览仪表盘待办

**Tab① 总览仪表盘的待办事项表格中新增待办类型**:

| 优先级 | 事项 | 来源 | 详情 |
|-------|------|------|------|
| 🟡 中 | 非标参数「平台高度=2.3m」已使用3次，建议转正 | 配置器 | [查看详情][一键转正] |
| 🔴 高 | 非标参数「护栏材质=碳纤维」已使用10次，强烈建议转正 | 配置器 | [查看详情][一键转正] |

点击 [查看详情] → 展开面板:
- 使用明细表格: 时间 | 项目 | 销售 | 完整配置方案
- 关联产品系列
- 当前超级BOM中最接近的标准值
- [创建ECO转正] [标记忽略(不再提示)]

### 5.3 ECO审批端 — Tab④

ECO审批流程中，非标转正的ECO有特殊展示:

- **触发源标签**: "CTO规则扩展"（紫色tag，区别于普通ECO）
- **额外信息区域**:
  - 非标参数使用统计（次数/项目/销售）
  - 成本影响预估（新增该选项对超级BOM的成本影响）
  - 供应链影响（是否需要新模具/新工艺）
- **审批通过后自动执行**:
  1. 将新参数值加入超级BOM规则的options/range
  2. plm_non_standard_param.status → PROMOTED
  3. plm_non_standard_param.promoted_eco_id → 关联ECO编号
  4. 通知所有曾使用该非标参数的销售："您之前申请的非标参数已转为标准配置，下次可直接在配置器中选择。"

### 5.4 驳回/忽略流程

**ECO驳回**:
- plm_non_standard_param.status → TRACKING（回到追踪状态，继续累计）
- 驳回理由记录到ECO
- usage_count继续累计，下次达到更高阈值(如5次)时再次触发

**管理员忽略**:
- plm_non_standard_param.status → DISMISSED
- 不再触发提示
- 管理员可在"非标参数管理"列表中重新激活

## 6. 非标参数管理列表（新增页面）

在Tab⑤配置器页面中，新增一个子Tab或底部面板："非标参数统计"

### 列表字段
| 列 | 说明 |
|----|------|
| 参数名 | param_key |
| 非标值 | param_value |
| 使用次数 | usage_count（可排序） |
| 首次使用 | first_used_at |
| 最近使用 | last_used_at |
| 关联项目数 | 去重后的project_name数量 |
| 状态 | TRACKING(追踪中) / PROMOTING(转正中) / PROMOTED(已转正) / DISMISSED(已忽略) |
| 操作 | [创建ECO转正] / [忽略] / [查看详情] |

### 筛选
- 状态筛选: 全部 / 追踪中 / 已达阈值(≥3) / 转正中 / 已转正 / 已忽略
- 默认: 显示"已达阈值"的记录

## 7. 数据流（端到端）

```
销售在配置器选配
  │
  ├─ 所有参数都在规则内 → 标准CTO流程（派生EBOM+报价）
  │
  └─ 存在非标参数 → 记录到 plm_config_log + 更新 plm_non_standard_param
       │
       ├─ usage_count < 3 → 走ETO流程，静默统计
       │
       └─ usage_count >= 3 → 触发转正建议
            │
            ├─ 销售/管理员点[转正] → 创建B级ECO
            │     │
            │     ├─ ECO审批通过 → 参数加入超级BOM规则 → status=PROMOTED
            │     │                → 通知相关销售
            │     │
            │     └─ ECO驳回 → status回到TRACKING → 继续累计
            │
            └─ 管理员点[忽略] → status=DISMISSED → 不再提示
```

## 8. 三级边界规则

```
✅ Always（必须做）:
- 每次配置器提交必须检查并记录非标参数（即使usage_count=1也要记录）
- 非标参数统计必须去重（同一user+project+param，24h内只计1次）
- 转正必须走B级ECO审批，不允许直接修改超级BOM规则
- ECO审批通过后自动更新超级BOM规则，不需要手工操作
- 转正阈值默认3次，可在系统设置中调整(2-10)

⚠️ Ask First（需确认再做）:
- L2(企微通知)和L3(企微+红色待办)的阈值是否需要调整（目前5次和10次）
- 转正后是否需要回溯历史订单（将已有的ETO订单标记为"可CTO化"）
- 是否需要按产品系列分别统计（目前是跨系列统计）

🚫 Never（禁止做）:
- 不允许自动转正（必须人工审批ECO）
- 不允许删除非标参数统计记录（审计追溯）
- 不允许在转正ECO审批期间（PROMOTING状态）再次创建转正ECO
- 非标参数提示不允许阻塞销售操作（提示后销售可继续走ETO流程）
```

## 9. 成功指标

| 指标 | 目标 | 计算方式 |
|------|------|---------|
| CTO覆盖率 | 逐月提升 | CTO订单数 / (CTO+ETO)订单数 × 100% |
| 平均转正周期 | < 5个工作日 | 从usage_count达标到ECO审批通过的平均天数 |
| 非标参数识别率 | 100% | 被识别记录的非标参数 / 实际非标参数 |
| 转正采纳率 | > 60% | 被转正的非标参数数 / 达到阈值的非标参数数 |
