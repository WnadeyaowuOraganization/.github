### 2.4 操作列定义

| 操作 | 说明 | 显示条件 |
|------|------|---------|
| 详情 | 打开右侧900px详情抽屉 | 始终显示 |
| 修正 | 编辑项目信息，保留历史版本（旧值→新值记录） | 始终显示 |
| 分配 | 弹窗选择商务（显示工作量+区域关系），自动状态→assigned | unassigned/verified |
| 流转▾ | 下拉显示合法目标状态 | 有合法目标时 |
| ✓有效 | 标记为有效线索（现阶段吴耀确认，后期商务也可） | 未反馈时 |
| ✗无效 | 标记为无效线索+填原因 | 未反馈时 |
| 删除 | Popconfirm确认删除 | 始终显示 |

**已删除操作**: 催更新（项目矿场信息由系统自动实时更新）、重新评估（移到批量操作）

### 2.5 总分Hover评分详情

6维度评分（满分100）:
- 规模匹配 (30分)
- 时效性 (25分)
- 产品匹配 (25分)
- 竞争强度 (10分，反向）
- 信源质量 (10分)

Hover气泡显示：各维度进度条 + 匹配理由文字 + AI建议文字

### 2.6 真实性列

5层验证体系结果摘要：
- ✅ 高可信(绿) — 多源交叉验证通过
- ⚠️ 中等(橙) — 单信源或待复检
- ❌ 低可信(红) — 信息过期/验证失败

列下方显示简短论证摘要（如"3源交叉验证通过"）

### 2.7 投标倒计时

当 `mineCategory='bidding'` 时，分类列后显示倒计时Tag:
- 红: ≤3天
- 橙: ≤7天
- 蓝: ≤14天
- 灰: >14天或截止时间未知

### 2.8 分配弹窗

弹窗内容：
- 商务人员列表（从系统用户中筛选商务角色）
- 每人显示：姓名、当前跟进项目数、该项目区域熟悉度（★评分）、推荐标签
- 分配备注TextArea
- 确认后：状态→assigned + 记录分配日志

## 4. 状态机定义

### 矿场状态（11个）

同现有定义，详见原设计文档。

### 状态流转规则

```
unassigned  → verified / assigned / invalid
verified    → assigned / invalid
assigned    → contacted / dormant / invalid
contacted   → tracking / bid_preparing / lost / dormant
tracking    → bid_preparing / lost / dormant
bid_preparing → bid_submitted / lost
bid_submitted → won / lost
won         → dormant
lost        → unassigned / dormant
invalid     → unassigned
dormant     → unassigned / invalid
```

## 6. 数据库设计（新增表）

### wdpp_project_mine_edit_log（修正历史）
```sql
CREATE TABLE wdpp_project_mine_edit_log (
  id BIGINT AUTO_INCREMENT PRIMARY KEY,
  project_mine_id BIGINT NOT NULL,
  field_name VARCHAR(100) COMMENT '修正字段名',
  old_value TEXT COMMENT '旧值',
  new_value TEXT COMMENT '新值',
  reason VARCHAR(500) COMMENT '修正原因',
  edited_by VARCHAR(100) COMMENT '修正人',
  edited_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_project_mine_id (project_mine_id)
) COMMENT='项目矿场修正历史';
```

### wdpp_project_mine_source（多信源链接）
```sql
CREATE TABLE wdpp_project_mine_source (
  id BIGINT AUTO_INCREMENT PRIMARY KEY,
  project_mine_id BIGINT NOT NULL,
  source_type VARCHAR(50) COMMENT '信源类型',
  source_name VARCHAR(200),
  source_url TEXT,
  discovered_at DATETIME,
  create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_project_mine_id (project_mine_id)
) COMMENT='项目矿场信源链接';
```

### wdpp_project_mine_assign_log（分配记录）
```sql
CREATE TABLE wdpp_project_mine_assign_log (
  id BIGINT AUTO_INCREMENT PRIMARY KEY,
  project_mine_id BIGINT NOT NULL,
  assigned_by VARCHAR(100),
  assigned_to VARCHAR(100),
  remark VARCHAR(500),
  assigned_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_project_mine_id (project_mine_id)
) COMMENT='项目矿场分配记录';
```

### wdpp_project_counterpart 新增字段
