## 使用说明

**这个文件解决什么问题**：功能太多记不住，不知道做了什么、在做什么、下一步做什么。

**怎么用**：
1. **快速查找** — `Ctrl+F` 搜业务名或关键词
2. **Sprint 规划** — 看"Sprint"列，决定哪些提前/推迟/砍掉
3. **策略变更** — 在"策略备注"列记一句话，后续决策有据可查
4. **定期审计** — 每个 Sprint 结束后花10分钟过一遍，更新状态

**状态说明**：
- 📋 计划中 — Issue 已创建，未开始开发
- 🔨 开发中 — 编程CC正在处理
- ✅ 已上线 — Dev 环境可用
- ⏸️ 暂缓 — 有依赖或策略调整，暂不开发
- 🔍 待评估 — 上线后需评估使用情况，可能调整或废弃
- ❌ 废弃 — 不再需要

**实时查询命令**（需要细节时用）：
```bash
# 查某业务线全部 open Issue
gh search issues --label <标签> --state open --repo WnadeyaowuOraganization/wande-play --json number,title,labels

# 查某 Sprint 全部 Issue
gh search issues "Sprint-N" --state open --repo WnadeyaowuOraganization/wande-play --json number,title
```
