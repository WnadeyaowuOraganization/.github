## 一、重组背景

### 1.1 现状问题

万德平台现有菜单存在以下问题：
1. **按内部部门组织**：商务部→支持中心→综合管理中心，这是B2B平台导航设计的头号反模式
2. **菜单混乱堆放**："万德业务"一级菜单下27个子项混乱堆放，无清晰分类
3. **重复冗余**："研发管控"和"运营工具"存在大量冗余功能
4. **角色混乱**：商务要看材质参数，设计师要查案例库，强制按角色分App导致频繁切换

### 1.2 核心决策（吴耀2026-04-06确认）

1. **不分App，分"工作区"（分组）**
2. **一个统一侧边栏，按工作场景分组**
3. **RBAC控制每个人能看到哪些分组**
4. **商务和设计共用功能，仅通过数据权限区分**

### 1.3 设计原则

参考Salesforce Lightning、纷享销客、SAP ERP Drawer导航：
1. 高频功能置顶，低频折叠
2. 同一功能不因角色拆分
3. 数据权限而非菜单权限控制差异
4. 支持个人级菜单自定义（拖拽排序+收藏夹）

## 六、验收清单

### 6.1 最终侧边栏验证

登录超管账号后，侧边栏应显示8个一级菜单，顺序为：
```
超管驾驶舱 → 商务部 → 支持中心 → 综合管理中心 → 安装售后中心 → 公共板块 → 资源中心 → 系统管理
```

### 6.2 功能验证

- [ ] 所有子菜单可点击，页面可正常加载
- [ ] 无404或白屏页面
- [ ] 旧一级菜单已隐藏，不再显示
- [ ] 无重复菜单项
- [ ] 不同角色登录看到对应权限的菜单
