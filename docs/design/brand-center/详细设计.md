### 2.2 Phase 1 · 基础设施

#### 2.2.1 账号 OAuth 绑定 (tab-oauth.html)
- **对应 Issue**: #1776（前端）/ #2145（后端）
- **组件**: 7 平台卡片网格 / 默认账号下拉 / 绑定历史表
- **7 平台**: LinkedIn / Twitter(X) / Facebook / YouTube / 微信公众号 / 视频号 / 抖音
- **数据源**:
  - `GET /wande/brand/accounts` — 所有绑定列表
  - `POST /wande/brand/accounts/{platform}/oauth` — 发起 OAuth
  - `DELETE /wande/brand/accounts/{id}` — 解绑
  - `PUT /wande/brand/accounts/{id}/default` — 设默认
- **代码现状**: 🔴 新建
- **三级边界**:
  - ✅ Always: OAuth token 过期前 7 天自动提醒；失败重试 ≤3 次
  - ⚠️ Ask First: 新增平台需评估合规与 API 稳定性
  - 🚫 Never: 不在前端存储 access_token；所有 token 走服务端代理

#### 2.2.2 角色权限 (tab-rbac.html)
- **对应 Issue**: #2201（fullstack，兼 biz:rbac）
- **4 级角色**: 管理员 / 编辑 / 审批 / 只读
- **组件**: 角色卡片 + 用户×角色矩阵 + 审计日志
- **数据源**: 复用平台 RBAC 体系 + 新增 `brand:*` 权限标识
- **代码现状**: 🟡 改造（平台 RBAC 已有，需注册品牌中心权限点）
- **权限点清单**:
  - `brand:center:view` / `brand:content:create/edit/publish` / `brand:approval:approve`
  - `brand:metrics:view` / `brand:account:bind` / `brand:case:manage`

#### 2.2.3 项目案例库 (tab-cases.html)
- **对应 Issue**: #1704（后端）/ #1727（前端）
- **组件**: 瀑布流 5×3 网格 + 筛选栏 + 详情抽屉
- **数据源**:
  - `GET /wande/brand/cases` 支持 `year/region/product/recommended/keyword` 筛选
  - `POST /wande/brand/cases` 创建
  - `PUT /wande/brand/cases/{id}/recommend` 推荐开关
  - 全文搜索走 ElasticSearch（或 MySQL FULLTEXT）
- **代码现状**: 🔴 新建
- **三级边界**:
  - ✅ Always: 封面图压缩后走 S3；案例编辑需审批
  - 🚫 Never: 未审批案例不得对外暴露

- **这是什么**: 万德全球品牌营销中枢，管理内容生产→分发→度量全链路
- **解决什么问题**:
  1. 多平台账号分散、难统一发布
  2. 内容生产效率低、AI 未充分利用
  3. 品牌声量无量化追踪、ROI 不清晰
- **快速上手**: (1) Phase1 绑定 7 平台账号 → (2) Phase2 用 AI 日历+数字人批量产出内容 → (3) Phase4 看健康度看板
