## §1 页面概述

### 1.1 体系定位

本设计是**架构级统一规范**，不是单一功能页面。核心目的：

1. **术语统一** — 消除"线索/商机/询盘/矿场项目"四个概念混用导致的团队困惑
2. **数据实体统一** — 新增 `leads` 表作为线索统一池，保留现有 `opportunities`、`inquiries`、`project_mine` 三张表
3. **流转规则统一** — 定义四入口 → 线索池 → 三类商机的自动/手动升级规则

**核心判断依据**（业内最佳实践 — Salesforce / HubSpot / Prospeo 2026）：
- **线索 Lead** = 未经验证的兴趣信号（举手）
- **商机 Opportunity** = 已确认的潜在交易（在谈具体一笔生意，有金额+关闭日期）
- **Opportunity Gate**（五项必须满足才能升级）：
  1. 在真实对话中确认了痛点需求
  2. 识别出了内部拥护者
  3. 聊过预算范围和时间节点
  4. 有下一步具体动作
  5. 预估金额 + 预计关闭日期

### 1.2 四入口归属

| 入口 | 来源 | 实体对应 | 升级策略 |
|------|------|---------|---------|
| 📧 外展获客 | #36 邮件/企微/LinkedIn | `leads` | 评分≥75 自动升级 |
| 📞 官网表单/展会名片 | 未来规划 | `leads` | 评分≥60 + 首次回访后升级 |
| 💬 客户询盘 | #39 经销/国贸 | `inquiries`（早期商机） | **不走 leads**，直接建 opportunity |
| ⛏️ 项目矿场 | #7 直销/B2G | `project_mine` | **手动确认**，系统只提示 |

**关键设计决策 D82**（2026-04-21）：
- 询盘本质是"早期商机"，已满足 Opportunity Gate 的前 3 项，不回退到线索
- 矿场项目本质是线索，但涉及投标流程的重大决策，保持手动确认
- 只有 外展+官网 走 `leads` 表 + 自动升级流程

### §2.3 评分引擎配置页（新增）

**组件**：`a-form` + `a-slider`（阈值调节）+ `a-switch`（规则开关）+ `a-input-number`（分数权重）

**布局**：
- 5 评分维度权重配置（ICP匹配度/参与度/预算信号/决策人/时间窗口）
- 3 升级规则开关：
  - 外展+询盘：评分≥75自动升级
  - 矿场：需手动确认（系统只提示）
  - 官网表单：评分≥60+首次回访后升级
- 负分规则配置（竞争对手/学生/求职者等）
- 休眠规则：N天无动作自动降级

**数据源**：
- `GET /api/crm/leads/scoring-rules` — 读配置
- `PUT /api/crm/leads/scoring-rules` — 更新（需 `crm:lead:scoring-rule:manage`）

**代码现状**：🔴 需新建 — 评分配置表 `lead_scoring_rules` 未创建

**三级边界**：
```
✅ Always:
  - 配置变更需写审计日志（who/when/before/after）
  - 权重总和校验 = 100%
  - 阈值调整需二次确认弹窗（影响自动化流程）

⚠️ Ask First:
  - 是否支持多套评分规则（按渠道不同）
```

## §3 API 契约

### §3.1 GET /api/crm/leads
**查询参数**：`channel, status, score_min, score_max, assigned_to, created_after, page, size`
**返回**：`{records: Lead[], total, pages}`

### §3.2 POST /api/crm/leads/{id}/convert
**请求体**：`{business_type: 'direct'|'dealer'|'international', initial_stage, expected_amount, expected_close_date}`
**返回**：`{opportunity_id, lead_status: 'CONVERTED'}`
**副作用**：
- leads 表 status → CONVERTED，写 converted_at
- 新建 opportunities 记录，关联 lead_id
- 写 crm_activity_log

### §3.3 GET /api/wande/project/mine/{id}/transit-readiness
**返回**：`{ready: bool, checks: {winProbability: {pass, value}, budgetConfirmed: {pass, evidence}, decisionMaker: {pass, name}}, missing: string[]}`

### §3.4 PUT /api/crm/leads/scoring-rules
**请求体**：`{weights: {...}, thresholds: {mql: 50, sql: 75}, autoUpgradeChannels: ['outreach', 'linkedin'], dormantDays: 30}`

## §4 数据库设计

### §4.1 新建表 `crm_leads`

```sql
CREATE TABLE crm_leads (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  name VARCHAR(100) NOT NULL COMMENT '联系人姓名',
  company VARCHAR(200) COMMENT '公司名',
  email VARCHAR(100),
  phone VARCHAR(30),
  source_channel VARCHAR(30) NOT NULL COMMENT 'OUTREACH_EMAIL/WECOM/LINKEDIN/WEBSITE/EVENT',
  source_detail VARCHAR(500) COMMENT '来源细节（序列ID/表单ID/展会名）',
  icp_match_score INT DEFAULT 0 COMMENT 'ICP匹配度 0-100',
  total_score INT DEFAULT 0 COMMENT '综合评分 0-100',
  status VARCHAR(20) NOT NULL DEFAULT 'NEW' COMMENT 'NEW/NURTURING/SQL/CONVERTED/DORMANT/DISQUALIFIED',
  assigned_to BIGINT COMMENT '跟进人 user_id',
  last_activity_at DATETIME,
  disqualification_reason VARCHAR(500),
  converted_opportunity_id BIGINT COMMENT '转化后 opportunity_id',
  converted_at DATETIME,
  create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
  update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  tenant_id VARCHAR(20) DEFAULT '000000',
  INDEX idx_status (status),
  INDEX idx_source (source_channel),
  INDEX idx_assigned (assigned_to),
  INDEX idx_score (total_score)
) COMMENT='线索统一池';
```

### §4.2 新建表 `crm_lead_scoring_rules`

```sql
CREATE TABLE crm_lead_scoring_rules (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  rule_key VARCHAR(50) UNIQUE,
  weights_json JSON COMMENT '{icp:30, engagement:25, budget:20, authority:15, timing:10}',
  mql_threshold INT DEFAULT 50,
  sql_threshold INT DEFAULT 75,
  auto_upgrade_channels JSON COMMENT '["OUTREACH_EMAIL","LINKEDIN","WECOM"]',
  dormant_days INT DEFAULT 30,
  negative_scoring_json JSON COMMENT '{competitor:-50, student:-25}',
  updated_by BIGINT,
  update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) COMMENT='线索评分规则';
```

### §4.3 新增字段（已有表）

- `crm_opportunity` 增加 `source_lead_id BIGINT NULL`（关联来源 lead，nullable=直接建的商机如询盘转来的）
- `project_mine` 增加 `converted_opportunity_id BIGINT NULL` + `transit_readiness_json JSON`

## §7 PageGuide 内容

**这是什么**：线索池是所有外展/官网/展会来源的早期潜在客户统一管理的地方。商机是已经在谈具体生意的记录。

**解决什么问题**：
- 之前"线索/商机/询盘/矿场"四个词混用，新员工搞不清
- 外展获客没有统一的转商机规则，评分≥75的好线索经常漏跟进
- 矿场项目转商机缺统一标准，导致销售预测失真

**快速上手**：
1. 新线索进入后看"线索池"（CRM → 线索管理）
2. 评分≥75分自动升级为商机（经销/国贸）或提示手动转（矿场）
3. 看架构蓝图（CRM → 架构蓝图）了解全流程
