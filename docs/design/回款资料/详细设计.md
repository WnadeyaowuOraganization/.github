## 1. 页面概述

- **路由**: `/wande/finance/payment-docs`
- **菜单层级**: 商务中心 > 回款资料管理
- **权限标识**: `finance:payment-docs:*`
- **所属模块**: `module:fullstack`
- **业务标签**: `biz:finance`
- **架构类型**: 多文件 3 Tab

### 业务背景

体游行业项目周期长（3-12个月），甲方回款按阶段触发，每个回款触发点都对应一批必须提交的资料。实际痛点：
- 资料缺失/延误直接导致回款延期，项目经理常在交付完成后才发现资料没齐
- 甲方要求的资料格式/清单因地区/行业/项目类型不同而千差万别，靠人工整理效率低
- 我司资质信息（营业执照/ISO证书/检测报告等）散落各处，填甲方表单需反复搜索

### 解决思路

**3 Tab 协同**：
1. **全阶段资料看板** — 主动推送「这个项目当前阶段缺什么资料」，避免交付后才发现
2. **甲方表单填写辅助** — 建立我司资质信息库，映射到甲方常见表单字段，一键填充
3. **甲方资料需求模板库** — 积累各地区/行业甲方的资料需求模板，新项目可匹配复用

### 2.1 全阶段资料看板（Dashboard）— 对应 Issue #3185

**原型文件**: `tab-dashboard.html`

- **组件**: 5 个 KPI 卡片 + 5 阶段完成度矩阵表 + 逾期资料 Top10 表格 + 催收触发按钮
- **数据源**:
  - `GET /api/payment-docs/dashboard/kpi` — 5 个 KPI 汇总
  - `GET /api/payment-docs/dashboard/stage-matrix` — 阶段×项目完成度矩阵
  - `GET /api/payment-docs/dashboard/overdue?limit=10` — 逾期资料 Top10
  - `POST /api/payment-docs/collect/trigger` — 触发企微催收
- **交互**:
  - 点击 KPI 下钻到对应筛选列表
  - 点击项目行展开 5 阶段资料完成度详情
  - 逾期资料点击「催收」→ 发起企微催收流程
- **代码现状**: 🔴 需新建（D68 新业务线，无历史代码）
- **已有文件**: 无

**5 个核心 KPI**:
| 指标 | 口径 | 用途 |
|------|------|------|
| 进行中项目数 | 当前 active 项目总数 | 总量参照 |
| 资料齐备率 | 已提交资料数 / 应提交资料数 × 100% | 健康度 |
| 逾期资料数 | 应提交日期已过但未提交的资料 | 风险量 |
| 本月回款触发点命中率 | 本月应触发回款数 / 实际触发回款数 | 业务成效 |
| 催收率 | 已发催收次数 / 逾期资料数 | 干预力度 |

**5 个阶段**（行向）:
1. 投标阶段（标书+报名资料）
2. 签约阶段（合同+付款条件+首款发票）
3. 生产阶段（生产进度报告+过程质检记录）
4. 安装验收阶段（发货清单+安装报告+验收单）
5. 质保期阶段（巡检记录+整改回单+质保到期资料）

### 2.2 甲方表单填写辅助（Template Autofill）— 对应 Issue #3189

**原型文件**: `tab-template.html`

- **组件**: 左侧「我司资质信息库」分类树 + 右侧「表单字段映射」编辑区 + 顶部「上传甲方表单」入口 + 底部「一键填充→导出」
- **数据源**:
  - `GET /api/payment-docs/our-profile` — 我司资质信息库（企业信息/证件/证书/人员/财务）
  - `POST /api/payment-docs/template/parse` — OCR 解析甲方表单结构
  - `POST /api/payment-docs/template/fill` — 字段映射后一键填充
  - `GET /api/payment-docs/template/field-map` — 历史字段映射库
- **交互**:
  - 上传甲方表单（PDF/Word/Excel）→ 后端 OCR/表格解析 → 展示识别的字段
  - 用户确认字段映射（自动匹配+手动修正）→ 保存到字段映射库
  - 一键填充后生成带公章位置标记的 PDF/Word
  - 支持中/英/法/西多语言填充
- **代码现状**: 🔴 需新建
- **已有文件**: 无

**我司资质信息库分类**（5 大类，≥50 字段）:
- 企业基本信息（公司全称/英文名/法人/地址/税号/电话/邮箱）
- 证件类（营业执照/组织机构代码证/税务登记证/外贸经营权）
- 资质/证书类（ISO9001/ISO14001/CE/TÜV/EN1176 检测报告）
- 关键人员信息（法人/技术负责人/质量负责人/外贸联系人）
- 财务/银行（银行账号/SWIFT Code/信用额度/注册资本）

### 2.3 甲方资料需求模板库（Client Req Library）— 对应 Issue #3191

**原型文件**: `tab-client-req.html`

- **组件**: 顶部三级筛选（地区/行业/项目类型）+ 中部模板列表卡片 + 右侧模板详情抽屉
- **数据源**:
  - `GET /api/payment-docs/client-req/templates?region=&industry=&projectType=` — 模板列表
  - `GET /api/payment-docs/client-req/templates/{id}` — 模板详情（包含资料 checklist）
  - `POST /api/payment-docs/client-req/templates` — 新增/维护模板（管理员）
  - `GET /api/payment-docs/client-req/match?projectId=` — 项目匹配推荐
- **交互**:
  - 按「华东/华南/中东/东南亚/北美...」× 「文旅/地产/政府/校园...」× 「体验馆/游乐场/主题公园...」三维筛选
  - 查看模板详情 → 显示资料 checklist + 每项样板文件 + 常见坑位
  - 新项目启动时 → 自动匹配最相似模板 → 一键应用到全阶段看板
- **代码现状**: 🔴 需新建
- **已有文件**: 无

**模板标签体系**:
- 地区维度：7 个一级区域（华东/华南/华北/西部/中东/东南亚/北美/欧洲/其他）
- 行业维度：文旅/地产/政府/校园/商业综合体/景区/幼教/民宿
- 项目类型：室内体验馆/户外游乐场/水上乐园/科技馆/无动力乐园/主题公园/攀爬馆/蹦床馆

## 3. API契约

### 3.1 Dashboard — §2.1

```
GET /api/payment-docs/dashboard/kpi
返回: { activeProjects, completionRate, overdueCount, triggerHitRate, collectRate }

GET /api/payment-docs/dashboard/stage-matrix?projectIds=
返回: [ { projectId, stages: [ { stage, required, submitted, overdue } ] } ]

GET /api/payment-docs/dashboard/overdue?limit=10
返回: [ { docId, projectId, projectName, stage, docName, dueDate, overdueDays, owner } ]

POST /api/payment-docs/collect/trigger
参数: { docIds: [], channel: 'wecom'|'email', remindLevel: 1|2|3 }
返回: { triggered, failedIds }
```

### 3.2 Template Autofill — §2.2

```
GET /api/payment-docs/our-profile
返回: { company:{}, certs:[], persons:[], finance:{} }

POST /api/payment-docs/template/parse
Body: multipart (file)
返回: { fields: [ { label, type, position, suggestedMapping } ] }

POST /api/payment-docs/template/fill
Body: { templateId, fieldMap: { fieldId: profileKey }, locale }
返回: { outputFileUrl }

GET /api/payment-docs/template/field-map
返回: [ { label, mappedKey, usageCount } ]
```

### 3.3 Client Req Library — §2.3

```
GET /api/payment-docs/client-req/templates?region=&industry=&projectType=
返回: [ { id, name, region, industry, projectType, docCount, usageCount } ]

GET /api/payment-docs/client-req/templates/{id}
返回: { meta, checklist: [ { stage, docName, required, sampleUrl, tips } ] }

POST /api/payment-docs/client-req/templates
Body: { name, region, industry, projectType, checklist: [] }

GET /api/payment-docs/client-req/match?projectId=
返回: [ { templateId, matchScore, reason } ]
```

## 4. 数据库设计

**表名前缀**: `wande_payment_doc_*`

### 4.1 新建表（6 张）

```sql
-- 项目资料要求（按 stage 拆行）
CREATE TABLE wande_payment_doc_requirement (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  project_id BIGINT NOT NULL,
  stage TINYINT NOT NULL COMMENT '1投标 2签约 3生产 4安装验收 5质保',
  doc_name VARCHAR(128),
  doc_category VARCHAR(64),
  required TINYINT DEFAULT 1,
  due_date DATETIME,
  source VARCHAR(32) COMMENT 'manual|template|auto',
  template_id BIGINT NULL,
  created_at DATETIME,
  INDEX idx_project_stage (project_id, stage)
);

-- 资料提交记录
CREATE TABLE wande_payment_doc_submission (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  requirement_id BIGINT NOT NULL,
  file_url VARCHAR(512),
  file_name VARCHAR(256),
  submitter_id BIGINT,
  submit_time DATETIME,
  status TINYINT COMMENT '1待审 2通过 3驳回',
  review_comment VARCHAR(512),
  INDEX idx_req (requirement_id)
);

-- 我司资质信息库（KV 表）
CREATE TABLE wande_payment_doc_our_profile (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  category VARCHAR(32) COMMENT 'company|cert|person|finance',
  field_key VARCHAR(64),
  field_value_zh TEXT,
  field_value_en TEXT,
  field_value_other JSON,
  attachment_url VARCHAR(512),
  expire_date DATETIME,
  updated_at DATETIME,
  UNIQUE KEY uk_field (category, field_key)
);

-- 甲方表单字段映射库
CREATE TABLE wande_payment_doc_template_field_map (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  source_label VARCHAR(256),
  mapped_profile_key VARCHAR(64),
  usage_count INT DEFAULT 0,
  last_used_at DATETIME,
  INDEX idx_label (source_label)
);

-- 甲方资料需求模板
CREATE TABLE wande_payment_doc_client_req_template (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  name VARCHAR(128),
  region VARCHAR(32),
  industry VARCHAR(32),
  project_type VARCHAR(32),
  doc_count INT,
  usage_count INT DEFAULT 0,
  created_by BIGINT,
  created_at DATETIME,
  INDEX idx_filter (region, industry, project_type)
);

-- 模板 checklist 明细
CREATE TABLE wande_payment_doc_client_req_item (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  template_id BIGINT NOT NULL,
  stage TINYINT,
  doc_name VARCHAR(128),
  required TINYINT,
  sample_file_url VARCHAR(512),
  tips TEXT,
  sort_order INT,
  INDEX idx_template (template_id)
);
```
