## 二、修复策略

### 2.1 问题分类

根据契约文件中的 FIXME 标注，问题分为三类：

1. **后端有，前端无调用** (最常见)
   - 管理后台API不需要前端调用
   - 前端页面缺失对应功能
   - **需创建Issue**: 补充前端页面或API调用

2. **前端有，后端未实现**
   - 功能开发中
   - 前端调用了错误路径
   - **需创建Issue**: 后端实现缺失API或前端修正路径

3. **路径/方法不匹配**
   - 前后端URL不一致
   - HTTP方法不一致
   - **需创建Issue**: 统一接口规范

### 2.2 修复优先级

按 Sprint1 重点模块排序：

| 优先级 | 模块 | 原因 |
|--------|------|------|
| P0 | execution | 大量新页面需要开发，接口必须先行对齐 |
| P0 | crm | 销售记录体系核心模块 |
| P0 | d3 | 设计模块AI功能依赖接口 |
| P1 | cockpit | 驾驶舱功能 |
| P1 | finance | 财务相关 |
| P2 | 其他 | 按需修复 |
